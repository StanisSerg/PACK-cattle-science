# C.5 — Resource Characterization

## 1. Зачем это читать

Если вы когда-нибудь видели, как на ферме планируют **закупку кормов**, учитывая только цену за тонну, но не сезонность, влажность, срок годности, логистику и совместимость с кормосмесителем — вы встретили C.5 «в диком виде».

> **FPF-тезис:** *«Ресурс должен быть охарактеризован не только количеством, но и качеством, доступностью, ограничениями, рисками и политикой использования. Планирование без характеризации ресурсов приводит к дефициту, порче или неправильному использованию.»*

C.5 — Resrc-CAL (Resource Characterization & Allocation Lens). Это паттерн для описания ресурсов: кормов, земли, воды, труда, оборудования, данных, времени. Ключевая идея: ресурс — это не просто «есть/нет», а профиль с многими измерениями.

**Фермерский пример:**

> Ферма закупает жмых подсолнечника. Покупают 50 тонн по низкой цене. Но не учли: влажность 14% вместо 12%, срок хранения 2 месяца, склад переполнен, кормосмеситель не может перемалывать крупные гранулы. Часть жмыха портится, часть требует дополнительной обработки. C.5 предписывает характеризовать ресурс до закупки.

---

## 2. История одной ошибки

Представьте ферму, которая планирует **рацион на лето**:

> «Кукурузный силоса хватит. У нас 200 тонн.»

Через месяц приходит аудитор. Он спрашивает:

- «200 тонн — это сухого вещества или как есть?» — Ответ: «Как есть» (но DM-%) важен).
- «Какая питательная ценность?» — Ответ: «Обычная» (но NDF, NEL, крахмал варьируются).
- «Какой срок годности?» — Ответ: «Ну, силос» (но качество падает со временем).
- «Как распределён по группам?» — Ответ: «Будем по мере надобности» (нет allocation policy).
- «А если поставщик задержит?» — Ответ: «Договоримся» (нет contingency).

Аудитор уходит ни с чем. Потому что ресурс не был охарактеризован. C.5 предлагает: состав, качество, доступность, ограничения, allocation, риски.

## 3. Resource Characterization — полное описание

### 3.1 Ресурсный профиль

**Ресурс = {kind, quantity, quality, availability, constraints, risks, policy}**

**На ферме для кукурузного силоса:**

| Атрибут | Значение | Тип |
|---|---|---|
| kind | corn silage | nominal |
| quantity_as_is | 200 t | ratio |
| DM | 35% | ratio |
| quantity_DM | 70 t DM | ratio |
| NEL | 6,5 MJ/kg DM | ratio |
| NDF | 45% | ratio |
| starch | 30% | ratio |
| pH | 3,9 | ratio |
| storage_life | 12 months | ratio |
| available_from | 2026-09-01 | date |
| available_until | 2027-08-31 | date |
| compatible_with | TMR mixer M1 | boolean |
| risk | mold if opened >7 days | risk record |

**Ключевой признак:** Количество без качества — бесполезно для планирования.

### 3.2 Качество и измеримость

**На ферме:**
- **Корма:** DM, NEL, CP, NDF, минералы, микотоксины.
- **Вода:** pH, жёсткость, содержание нитратов, доступность по часам.
- **Земля:** площадь, тип почвы, влажность, доступность сезонная.
- **Труд:** количество людей, квалификация, смены, переработки.
- **Оборудование:** производительность, простои, обслуживание.

**Ключевой признак:** Каждый ресурс должен иметь CHR-измерения.

### 3.3 Availability и scheduling

**На ферме:**
- Силос доступен с сентября по август.
- Свежая трава — только в сезон.
- Ветеринар — по графику, в выходные — дежурный.
- Доильный робот — 22 часа/сутки, 2 часа на мойку.

**Ключевой признак:** Availability — это не «есть», а функция времени и условий.

### 3.4 Constraints

**На ферме:**
- **Жёсткие:** вместимость силосной ямы; мощность кормосмесителя; нормы содержания животных.
- **Мягкие:** предпочтительный поставщик; целевой уровень mycotoxin; бюджет.

**Ключевой признак:** Constraints — это gates для allocation decisions.

### 3.5 Allocation policy

**На ферме:**
- Высококачественный силос → высокопродуктивные коровы.
- Силос с повышенным NDF → сухостойные.
- Вода с высоким содержанием нитратов → не для телят.
- Ветеринарное время в пик отёлов → резервируется заранее.

**Ключевой признак:** Allocation — это не просто «раздать», а применение policy с учётом приоритетов.

### 3.6 Contingency и риски

**На ферме:**
- Риск: поставщик задерживает жмых → contingency: запас 2 недели.
- Риск: силосная яма протекает → contingency: резервная ёмкость.
- Риск: перебой с водой → contingency: резервуар.

**Ключевой признак:** Риски ресурса должны быть явными и иметь mitigation plan.

## 4. Каскад типичных ошибок

Возьмём снова:

> *«Кукурузного силоса хватит. У нас 200 тонн.»*

**Разложение по C.5:**

| Часть утверждения | Что это в FPF | Почему ошибка |
|---|---|---|
| «200 тонн» | **quantity_as_is** | Без DM, качества, срока |
| «хватит» | **Неявная availability** | Нет расписания потребления |
| «у нас» | **Нет spatial/condition info** | Где хранится, в каком состоянии |
| «силос» | **Нет quality profile** | NEL, NDF, pH — неизвестны |

**Что плохого в смешивании:**

1. **Неправильный рацион.** Без DM и NEL невозможно сбалансировать питание.
2. **Порча.** Силос может испортиться, если не учтён срок годности.
3. **Дефицит.** 200 тонн «как есть» может быть 70 тонн DM — меньше, чем нужно.

## 5. Как это выглядит на ферме: правильное применение

Перепишем планирование силоса.

**Было (смешанное):**
> «Кукурузного силоса хватит. У нас 200 тонн.»

**Стало (разложенное):**

**Resource profile:**
```
ResourceID: corn_silage_bunker_A_2026
kind: corn silage
quantity_as_is: 200 t
DM: 35%
quantity_DM: 70 t
NEL: 6,5 MJ/kg DM
NDF: 45%
starch: 30%
pH: 3,9
storage_life: 12 months
available: 2026-09-01 → 2027-08-31
location: bunker_A, opened 2026-09-15
compatible: TMR mixer M1
risks:
  - mold if face exposed >7 days
  - heating if DM <30%
contingency: use haylage if spoilage >10%
```

**Allocation policy:**
```
Group              | Daily DM allocation | Rationale
-------------------|---------------------|----------------------------------
High-yielders      | 8 kg DM/cow         | Highest NEL need
Mid-lactation      | 6 kg DM/cow         | Balanced energy
Dry cows           | 3 kg DM/cow         | Lower energy, fiber need
```

**Что это даёт:**
- Понятно, сколько реально сухого вещества.
- Качество силоса учтено в рационах.
- Риски и contingency явны.
- Allocation соответствует потребностям групп.

## 6. Практическое применение: с чего начать

**Шаг 1.** Составьте список критических ресурсов: корма, вода, земля, труд, оборудование, данные.

**Шаг 2.** Для каждого ресурса создайте профиль: kind, quantity, quality, availability, constraints.

**Шаг 3.** Всегда указывайте DM/концентрацию, если ресурс неоднороден.

**Шаг 4.** Задайте allocation policy: кто/когда/сколько получает.

**Шаг 5.** Выпишите риски и contingency для каждого ресурса.

## 7. Проверь себя

| Вопрос | Если ответ «да» — проблема |
|---|---|
| Планируете ли по quantity_as_is? | Нет DM/качества |
| Знаете ли срок годности ресурса? | Риск порчи |
| Есть ли allocation policy? | Хаотичное распределение |
| Учтены ли seasonal constraints? | Availability не как функция |
| Есть ли contingency plan? | Отсутствие resilience |
