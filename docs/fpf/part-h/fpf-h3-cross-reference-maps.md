# H.3 — Cross-Reference Maps: карты перекрёстных ссылок

> **Цель capture:** объяснить, как двунаправленные карты ссылок связывают термины из разных частей FPF, и почему они нужны для перехода от понятия в одной части к управляющему паттерну в другой.

---

## 1. Зачем это читать

FPF разбит на части A–K, и одно понятие может обсуждаться в нескольких из них с разными акцентами. В части A понятие может быть введено как базовый тип, в части C — как процессное отношение, в части B — как требование к доказательствам. Если специалист видит только одну часть, он рискует применить локальное определение вне его области действия. Cross-Reference Maps устраняют этот разрыв.

> **FPF-тезис:** *«Понятие без двунаправленной ссылки остаётся в ловушке одной части.»*

**Фермерский пример:**

> На ферме обсуждают «границу отёла». Для зоотехника это временной интервал вокруг отёла (часть C). Для ветеринара — физическое пространство родильного отделения (часть A). Для экономиста — раздел учёта между двумя лактациями (часть B). Без карты ссылок каждый остаётся при своём понимании, и протоколы противоречат друг другу.

---

## 2. История одной ошибки

В хозяйстве внедрили систему учёта переходного периода. Зоотехнический отдел определил «transition period» как интервал от 21 дня до отёла до 21 дня после отёла по C.26. Ветеринарный отдел использовал то же слово для физического загона подготовленных коров по A.6.B. При синхронизации данных выяснилось, что «корова в transition period» в одной системе означает временной интервал, а в другой — местоположение. Отчёты оказались несопоставимы.

---

## 3. Cross-Reference Maps — полное описание

### 3.1 Бидирекционная связь

**Определение.** Cross-Reference Map — это таблица связей, в которой каждый термин одной части FPF сопоставлен с соответствующим термином или паттерном в другой части, и обратно.

**Пояснение.** Связь двунаправленная: из части A можно попасть в часть C, и из части C — обратно в A. Это важно, потому что пользователь может начинать с любой точки. Карта не заменяет определения, но показывает, где искать управляющий паттерн.

**Пример из животноводства.** Термин «boundary» встречается в A.6.B (`s.SpatialBoundary`), A.6.C (`s.ScopeBoundary`) и C.26 (`s.ProcessBoundary`). Карта показывает, что для физического загона нужен A.6.B, для области действия утверждения — A.6.C, для временного интервала — C.26.

**Ключевой признак.** Если один термин используется в разных частях спецификации, он должен быть представлен в Cross-Reference Map.

### 3.2 Управляющий паттерн

**Определение.** Управляющим для термина является тот паттерн, который задаёт его текущую семантику и регулирует его применение в решениях.

**Пояснение.** Один термин может иметь несколько связанных паттернов, но только один из них управляет в конкретном контексте. Cross-Reference Map помогает определить, какой паттерн взять за основу.

**Пример из животноводства.** Термин «risk» связан с C.2 (классификация риска), C.3 (клинический риск) и C.5 (экономический риск). Если речь идёт о BHB-скрининге, управляющим является C.2; если о ветеринарном прогнозе — C.3; если об убытках — C.5.

**Ключевой признак.** Для каждого использования термина в документе должен быть указан управляющий паттерн.

### 3.3 Карта Part A ↔ Part C ↔ Part B

**Определение.** Триада Part A ↔ Part C ↔ Part B — это основная структура перекрёстных ссылок: структурные типы (A), процессы и отношения (C), доказательные и нормативные требования (B).

**Пояснение.** Part A отвечает на вопрос «из чего состоит». Part C — «как это происходит». Part B — «как это обосновать». Карта позволяет двигаться между этими уровнями: от коровы (A) к её metabolic transition (C) к требованиям доказательности риска (B).

**Пример из животноводства.** `s.Individual` (A.1.1) → `hasState` → `s.MetabolicState` (C.21) → `evidencedBy` → `s.Observation` (B.3). Карта показывает, что утверждение о метаболическом состоянии должно быть обосновано наблюдением.

**Ключевой признак.** Если аргумент проходит через A → C → B, каждый переход должен быть поддержан relation из карты.

## 4. Почему смешивать / игнорировать — значит рисковать

Рассмотрим типичное смешанное утверждение:

> *«Риск кетоза основан на границе стада.»*

**Разложение по H.3:**

| Часть утверждения | Что это в FPF | Почему важно разделять |
|---|---|---|
| «риск кетоза» | C.2 / C.3 | Классификация или клинический риск |
| «основан на» | relation `evidencedBy` / `derivedFrom` | Разные relations требуют разных доказательств |
| «граница стада» | A.6.B / A.6.C / C.26 | Пространство, область или интервал |

---

## 5. Как это выглядит на ферме: правильное применение

**Было (смешанное / нечёткое):**

> В SOP написано: «Оценивать риск внутри стада». Никто не уточнял, «стадо» — это физическая группа, учётная единица или популяция для анализа. Протоколы отделов зоотехники и аналитики расходились.

**Стало (разложенное / ясное):**

> В SOP три варианта: «физическое стадо → A.6.B SpatialBoundary»; «учётная единица → A.6.C ScopeBoundary»; «аналитическая популяция → C.26 ProcessBoundary». Каждый вариант связан с управляющим паттерном и relation.

---

## 6. Практическое применение: с чего начать

1. При появлении неоднозначного термина откройте Cross-Reference Map.
2. Найдите все части FPF, где встречается этот термин.
3. Определите, какой паттерн управляет вашим контекстом.
4. Зафиксируйте выбранный паттерн и relation в документе.
5. При изменении контекста проверьте, не сменился ли управляющий паттерн.

---

## 7. Проверь себя

| Вопрос | Если ответ «да» — проблема |
|---|---|
| Один термин используется в документах разных отделов по-разному? | Нет общей карты ссылок |
| Вы не можете найти управляющий паттерн для термина? | Риск локального определения |
| Ссылки между частями A, B, C отсутствуют или однонаправлены? | Карта неполная |
| При смене контекста не меняется интерпретация термина? | Возможно нарушение границ применимости |

## 9. Что запомнить

1. Cross-Reference Map связывает термины из частей A, B и C двунаправленно.
2. Для каждого использования термина должен быть выбран управляющий паттерн.
3. Один термин в разных частях может иметь разную семантику.
4. Карта не заменяет определения, но указывает, где их искать.
5. При переходе между контекстами проверяйте соответствие relation и паттерна.
