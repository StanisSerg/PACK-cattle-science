# Шпаргалка: эпистема — определение и виды

> Быстрая справка для определения, является ли что-то эпистемой, и какой она бывает.

---

## Определение

**Эпистема (episteme)** — это **знание с claim (утверждением)**.

Три обязательных элемента:
1. **Объект** (EntityOfConcern) — о чём утверждение.
2. **Claim** — что именно утверждается.
3. **Контекст / scope** — где, когда и при каких условиях это верно.

Если одного из трёх нет — это не эпистема.

---

## Быстрый тест

| Вопрос | Если ответ «да» |
|--------|-----------------|
| «О чём это утверждает?» | Возможно, эпистема. |
| «На чём это основано?» | Возможно, evidence или measurement. |
| «В каком контексте это верно?» | Возможно, bounded-context claim. |

Если на все три вопроса есть ответ — перед тобой эпистема.

---

## Эпистема vs не-эпистема

| Это | Пример | Почему не эпистема |
|-----|--------|--------------------|
| **System / holon** | Корова №4523 | Физическая сущность, не claim. |
| **Data** | `BHB = 0,9` | Сырое число без интерпретации. |
| **Carrier** | PDF-отчёт, dashboard tile, фото | Носитель, внутри которого может быть эпистема. |
| **Event / utterance** | Звук клика, слово «кетоз» | Событие или звук без claim. |

## Основные виды эпистем

| Вид | Что утверждает | Вопрос для определения | Пример |
|-----|----------------|------------------------|--------|
| **Description** | Какой объект / состояние | «Какой объект? Какое состояние?» | «Корова №4523 — субклинический кетоз.» |
| **Measurement** | Измеренная характеристика | «Какое значение, как измерено, по какой шкале?» | «BHB = 0,9 ± 0,05 ммоль/л, фотометрия.» |
| **MethodDescription** | Как что-то делать | «Как выполнить? По какому рецепту?» | «Рецепт AMTS: задать CP → NDF → проверить по NASEM.» |
| **Method** | Абстрактный способ | «Какой общий подход?» | «Метод оценки риска по BHB и NEFA.» |
| **Dynamics** | Как система меняется | «Что будет, если…?» | «При снижении DCAD риск ацидоза падает.» |
| **Commitment** | Что должно быть | «Кто обязан что сделать?" | «Ветеринар должен осмотреть за 2 часа.» |
| **PromiseContent** | Что обещается | «Что роль обещает потребителю?" | «Точность прогноза 85% при ежедневных данных.» |
| **Evidence** | На чём основано утверждение | «Что подкрепляет claim?" | «BHB 0,9 + DIM 35 + калибровка X.» |
| **Work record** | Что фактически произошло | «Кто, когда, по какому методу, с каким результатом?" | «Ветеринар осмотрел корову в 14:30, выявил кетоз.» |
| **WorkPlan** | Что планируется | «Что собираемся сделать?" | «Рекомендуется осмотр ветеринара.» |
| **Language-state position** | Стадия зрелости описания | «Насколько это утверждение зрелое?" | Sketch → Candidate → Settled → Retired. |
| **Pre-articulation cue** | Нечёткое наблюдение для проверки | «Это worth attention?" | «После смены корма коровы чаще лежат — стоит проверить.» |

## Boundary Norm Square — L/A/D/E

Граничные утверждения бывают четырёх видов:

| Класс | Что утверждает | Пример |
|-------|----------------|--------|
| **L — Laws** | Закон / факт / инвариант | «Корова — млекопитающее.» |
| **A — Admissibility** | Что допустимо | «Доступ к медкарте только ветеринару.» |
| **D — Deontics** | Обязательства / запреты | «Ветеринар должен проверить температуру.» |
| **E — Work-Effects** | Эффект от Work | «После вакцинации статус — immune.» |

---

## Language-state шкала зрелости

| Состояние | Описание | Пример |
|-----------|----------|--------|
| **Sketch** | Эскиз, идея, предположение | «Может, DCAD влияет на ацидоз?» |
| **Candidate** | Проверяемая гипотеза | «При снижении DCAD риск ацидоза падает.» |
| **Settled** | Принятое в контексте знание | «NASEM 2021: DCAD и ацидоз связаны вот так.» |
| **Retired** | Снятое с доверия | «Старая модель больше не применима.» |

---

## Типичные ошибки

| Ошибка | Почему неверно | Как исправить |
|--------|----------------|---------------|
| «Данные = эпистема» | Data без интерпретации не несёт claim. | Добавить метод, шкалу, интерпретацию. |
| «Модель = реальность» | Модель — episteme, system — физика. | Явно разделять model и actual system. |
| «Рекомендация = действие" | WorkPlan ≠ Work. | Проверять, выполнено ли. |
| «Дашборд = evidence" | Dashboard tile — publication face. | Искать register-backed source. |
| «Эпистема всегда верна" | Эпистема — claim, может быть ошибочной. | Проверять evidence и scope. |

## Пример разложения

> «Decision Layer рекомендует осмотр коровы №4523 из-за высокого риска кетоза.»

| Часть | FPF-классификация |
|-------|-------------------|
| Корова №4523 | System / EntityOfConcern |
| Данные BHB, DIM | Data → Evidence |
| Модель риска кетоза | Episteme (Dynamics / MethodDescription) |
| «Высокий риск кетоза» | Episteme (Description) |
| «Рекомендуется осмотр» | WorkPlan |
| Текст alert | Publication face |
| Осмотр ветеринаром | Work |

---

## Итоговая формула

> **Эпистема = объект + claim + контекст.**

Если claim связан с измерением — добавляется **scale**. Если claim о должном — это **Commitment**. Если о том, как делать — **MethodDescription**. Если о том, как меняется — **Dynamics**.
