# Эпистема vs не-эпистема: примеры

> **Правило:** эпистема — это знание, которое несёт **claim** (утверждение) и может быть проверено, оспорено или использовано как evidence (доказательство). Не всякие данные, объекты или события сами по себе являются эпистемами.

| Пример | Эпистема? | Почему | Что нужно добавить, чтобы стало эпистемой |
|--------|-----------|--------|--------------------------------------------|
| **Корова №4523** | ❌ Нет | Это **system** (система) / holon (холон), физическая сущность. Сама по себе она не делает утверждений. | Описание, диагноз, измерение — тогда получится эпистема о корове. |
| **«BHB = 0,9 ммоль/л»** | ❌ Нет | Сырое значение без интерпретации — это **data** (данные), не claim. | Указать контекст: «BHB = 0,9 ммоль/л, калибровка X, при пороге > 0,8 свидетельствует о субклиническом кетозе». |
| **«У коровы №4523 субклинический кетоз»** | ✅ Да | Это **Description** (описание) — claim о состоянии EntityOfConcern. | — |
| **Модель AMTS рациона** | ✅ Да | Это **episteme** — знание о том, какой ожидается результат при заданном рационе. | — |
| **Корм в кормушке** | ❌ Нет | Физический объект, часть system. Сам по себе не несёт claim. | Анализ состава корма + метод измерения + интерпретация → эпистема о корме. |
| **Рецепт расчёта рациона (NASEM/AMTS)** | ✅ Да | Это **MethodDescription** — знание о том, как делать. | — |
| **Запись «ветеринар осмотрел корову в 14:00»** | ⚠️ Граничный случай | Это **Work** (работа / факт) — запись о событии. Сама по себе не claim о состоянии системы, но может служить evidence. | Добавить результат осмотра: «осмотр выявил субклинический кетоз» — тогда это эпистема-диагноз. |
| **Код Decision Layer** | ⚠️ Зависит | Как статический текст — это **MethodDescription** (эпистема). Как запущенный процесс — это **system**. | Нужно понимать, в каком контексте рассматриваем. |
| **Плитка дашборда «кетоз — зелёно»** | ❌ Нет | Это **publication face** (публикационная поверхность). Без register-backed source (регистрового источника) — просто картинка. | Ссылка на исходную эпистему: лабанализ, модель, ветеринарный диагноз. |
| **«Ветеринар обязан осмотреть корову в течение 2 часов»** | ✅ Да | Это **Commitment** (обязательство) — deontic-claim о должном. | — |
| **PDF-отчёт лаборатории** | ❌ Нет | Это **carrier** (носитель) — файл, в котором могут быть эпистемы. | Сам PDF не делает claim; прочитанные и интерпретированные данные внутри него — эпистемы. |
| **«Если BHB > 0,8 → alert»** | ✅ Да | Это **decision rule** (правило решения) — claim о том, какой вывод делать при каком условии. | — |
| **Звук клика по кнопке «сохранить»** | ❌ Нет | Это событие / utterance (высказывание) без интерпретации. | Запись в журнале Work: «пользователь сохранил рацион в 14:05» — тогда это факт-запись. |
| **Уравнение NASEM для переваримости** | ✅ Да | Это **Dynamics** / **MethodDescription** — reusable знание о закономерности. | — |
| **Git commit `a1b2c3d`** | ❌ Нет | Это **identifier** (идентификатор) / carrier. Сам хеш не несёт claim о состоянии коровы. | Commit message + diff + контекст: «в этом коммите обновлена модель кетоза» — тогда commit становится evidence для claim. |

## Ключевые сигналы

**Это эпистема, если можно спросить:**
- «О чём это утверждает?» — и есть ответ.
- «На чём основано?» — и есть evidence или grounding.
- «В каком контексте это верно?» — и есть scope.

**Это не эпистема, если:**
- это физический объект;
- это сырое значение без интерпретации;
- это carrier (файл, дашборд, хеш, utterance) без явного claim;
- это просто событие без трактовки.
