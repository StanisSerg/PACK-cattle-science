# Wording-Use Ontological Precision Restoration (E.10.ARCH) для семинара FPF-02

## Триггерные строки и их восстановление

| Триггерная формулировка | Скрытая онтологическая проблема | Восстановленная формулировка |
|-------------------------|----------------------------------|------------------------------|
| "Система весит два килограмма" | Путаница системы и её описания | "Описание системы, представленное на бумаге, весит два килограмма; сама система — это физический объект" |
| "Архитектура холодильника" | Архитектура vs архитектурное описание | "Архитектурное описание холодильника; архитектура — в самом холодильнике" |
| "Программа работает" | Код vs действующая система | "Действующая программа как физическая система выполняет метод, описанный в коде" |
| "ФПФ говорит" | Агентификация эпистемы | "Паттерны FPF указывают на необходимость..." |
| "Результат получен" | Неясный тип "результата" | "Объект X получен как результат отношения Y" |
| "Мы делаем проект" | Проект как работа vs проект как сеть потоков | "Проект — это сеть потоков преобразования, в которой мы выполняем работы" |
| "Это важно" | Не указано, для чего важно | "Это важно для [характеристики/решения], потому что..." |
| "Агент сделал" | Не указана роль агента | "Агент в роли [экзекутора/ревьюера/платформенного инженера] выполнил..." |
| "Структура чего?" | Неявный тип отношений | "Структура [функциональная/модульная/конструктивная] системы X" |
| "Улучшим доступ пациента" | Скрытый kind | "Изменим [назначение на приём / интерфейс записи / маршрут лечения] для пациента" |

## Архитектура восстановления

```
E.10 (recognition) → E.10.ARCH (architecture selection) → direct governing pattern (C.30.P, A.6.P, A.16 и т.д.)
```

Для семинара FPF-02 основной движущей силой является A.6.P (RPR), потому что большинство проблем — в размытых отношениях.

## Рекомендации по применению

- При чтении семинара помечать каждую фразу, где слово может относиться к разным kind.
- Возвращаться к FPF для проверки, какой kind у термина в данном контексте.
- В агентских запросах использовать восстановленные kind-explicit формулировки.

## Boundary

E.10.ARCH выбирает архитектуру восстановления, но не выполняет восстановление самостоятельно. Реализация — C.30.P, A.6.P и другие управляющие паттерны.
