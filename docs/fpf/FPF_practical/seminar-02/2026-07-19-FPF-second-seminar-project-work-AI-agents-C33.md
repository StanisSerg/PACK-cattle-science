# Structural Information Adequacy (C.33) для NARRATIVE семинара FPF-02

## Захваченные структуры (captured structure)

| Структура | Где в narrative | Примечание |
|-----------|-----------------|------------|
| Система как физический объект | Блок 6 | Сохранено различение система/описание |
| Граница системы | Блок 6 | Первый взгляд наружу/вверх |
| Холон и метахолонный переход | Блок 7 | Масштаб vs уровень, флот, печень/организм |
| Целевая система как роль | Блок 8 | assign, чашка, код vs программа |
| Потоки преобразований | Блок 9 | констрактор, кейс, build the builder |
| Мантры | Блок 10 | Автоматизм, история с вождением |
| Характеристики и архитектурный кандидат | Блок 11 | Гудхарт, выбор структур |
| Содержание vs платформенная инженерия | Блок 12 | Agile Manifesto, баланс |
| Агентский контур | Блок 3 | экзекутор, ревьюер, законы Амдала/Конвея |

## Потерянные структуры (lost structure / missing structure)

| Структура | Почему потеряна | Return condition |
|-----------|-----------------|------------------|
| Точные временные метки каждой реплики | Убраны для связности | Вернуться к исходному транскрипту |
| Имена участников и их вопросы | Не несут смысловой нагрузки для narrative | Вернуться к транскрипту |
| Подробные технические детали создания слайдомента | Слайдомент не целевой объект изучения | Вернуться к транскрипту или слайдоменту |
| Цветовая кодировка слайдов | Визуальная структура | Открыть слайдомент |
| Порядок практикумов и их детали | Coarsened для теории | Вернуться к слайдоменту |

## Оценка adequacy

Для declared architecture use (изучение и применение FPF) narrative adequate. Для цитирования или воссоздания семинара — inadequate; нужен source-bearing side.

## Boundary

C.33 не заменяет выбор архитектурного кандидата, публикацию записи, измерение значений или авторизацию работы.
