# EvaluationCharacteristicSpaceSpec: система обучения vs контрольная система vs саморазвитие

> **Назначение:** зафиксировать координаты, шкалы, правила оценки и защищённые компромиссы для сравнения трёх вариантов. Результат будет использоваться для `ChoiceResult` (C.11) или для решения о необходимости дополнительного пилота/сбора данных.
> **Статус:** черновик — совместное заполнение с пилотом в процессе.

---

## 1. ObjectSet (варианты для сравнения)

| ID | Вариант | Краткое описание |
|----|---------|------------------|
| A | **Создать систему обучения технологов** | Разработать repeatable метод/процесс, материалы и контрольные точки, позволяющие технологам применять методы PACK-cattle-science без постоянного участия пилота. |
| B | **Создать систему контроля с автоматической обработкой отчётов** | Внедрить инструменты сбора данных, отчётности и автоматической обработки, чтобы пилот получал информацию и мог вмешиваться без необходимости обучать технологов глубоко. |
| C | **Сосредоточиться на собственном развитии** | Инвестировать время в углубление личной экспертизы, решать задачи технологов по запросу, не масштабируя обучение или контроль формально. |

---

## 2. EvaluationPurpose (цель сравнения)

> Выбрать стратегию передачи и контроля методов PACK-cattle-science на текущем этапе: формализованная система обучения, контрольная система с автоматической обработкой, или личная экспертиза пилота как ограниченный ресурс.

**Ключевой вопрос:**
> Какой вариант лучше удовлетворяет текущие ограничения по времени, масштабу, риску и качеству, не смешивая personal capability с organizational capability?

---

## 3. CharacteristicFamilies (семейства характеристик)

| # | Характеристика | Зачем важна | Вес (TBD) |
|---|----------------|-------------|-----------|
| 1 | **Время до первого результата** | Сколько времени пройдёт, прежде чем выбранный вариант начнёт снижать нагрузку на пилота или повышать качество работы технологов. | |
| 2 | **Масштабируемость** | Сколько технологов можно охватить без пропорционального роста времени пилота. | |
| 3 | **Качество передачи знаний** | Насколько глубоко и точно метод доходит до технологов, включая тонкие контекстные решения. | |
| 4 | **Риск ошибки применения** | Вероятность и серьёзность ошибки, если технолог применит метод неправильно. | |

| 6 | **Стоимость ошибки для бизнеса** | Финансовые, операционные или репутационные потери от неправильного решения технолога. | |
| 7 | **Обратимость решения** | Насколько легко отказаться от выбранного варианта, если он не работает. | |

## 4. ScalesAndComparators (шкалы и компараторы)

Для каждой характеристики — единая шкала, применимая ко всем вариантам.

| Характеристика | Шкала | Вариант A (обучение) | Вариант B (контроль) | Вариант C (саморазвитие) | Источник оценки | Неопределённость |
|----------------|-------|----------------------|----------------------|--------------------------|-----------------|------------------|
| 1. Время до первого результата | Короткое / Среднее / Долгое | Среднее | Короткое | Долгое / неприменимо | Оценка пилота | Низкая для B; средняя для A и C |
| 2. Масштабируемость | Низкая / Средняя / Высокая | Высокая | Средняя | Низкая | Оценка пилота | Низкая |
| 3. Качество передачи знаний | Поверхностное / Среднее / Глубокое | Глубокое | Поверхностное | Глубокое, но не формализованное | Оценка пилота | Средняя |
| 4. Риск ошибки применения | Высокий / Средний / Низкий | Средний | Низкий | Низкий | Оценка пилота | Средняя |
| 5. Автономность от пилота | Нет / Частичная / Полная | Полная | Частичная | Нет | Оценка пилота | Низкая |
| 6. Стоимость ошибки для бизнеса | Высокая / Средняя / Низкая | Неясно | Неясно | Неясно | Пилот не смог привести конкретные примеры | Высокая |
| 7. Обратимость решения | Низкая / Средняя / Высокая | Низкая | Средняя | Высокая | Уточнение пилота | Низкая |
| 8. Доступность доказательств | Мало / Частично / Достаточно | Примерно равно | Примерно равно | Примерно равно | Оценка пилота | Средняя |

## 5. ProtectedTradeOffs (защищённые компромиссы)

> Что нельзя обменивать друг на друга напрямую, чтобы сравнение не искажалось.

| Нельзя менять | Почему |
|---------------|--------|
| Личное время пилота ↔ масштабируемость | Личная экспертиза не становится автоматически масштабируемой, пока не формализована. |
| Скорость передачи ↔ глубина знаний | Быстрый курс может дать поверхностное понимание, которое повышает риск ошибки. |
| Качество обучения ↔ риск ошибки | Нельзя считать, что «обучили» автоматически значит «будут применять правильно». |
| Время сейчас ↔ время потом | Создание системы требует инвестиций сейчас, но может экономить время позже; эти моменты времени неравнозначны. |

## 6. EvidenceRules (правила доказательства)

> Что может изменить оценку по каждой характеристике.

| Характеристика | Доказательство, которое повысит уверенность | Как получить |
|----------------|-----------------------------------------------|--------------|
| 1. Время до первого результата | Пилот системы обучения с одним технологом | Провести пилот за 1–2 недели |
| 2. Масштабируемость | Сколько технологов может пройти обучение параллельно | Смоделировать загрузку или провести групповой пилот |
| 3. Качество передачи знаний | Результаты контрольных точек / кейсов | Встроить проверку понимания в пилот |
| 4. Риск ошибки применения | Статистика ошибок до и после обучения | Сравнить периоды «без системы» и «с системой» |
| 5. Автономность от пилота | Число решений, принятых технологом без консультации | Зафиксировать обращения к пилоту до/после |
| 6. Стоимость ошибки для бизнеса | Конкретные случаи финансовых потерь | Собрать кейсы ошибок в кормлении/диагностике |
| 7. Обратимость решения | Оценка затрат на сворачивание системы | Просчитать минимальный комплект материалов и процессов |
| 8. Доступность доказательств | Наличие пилотных данных | Провести пилот или найти аналогичный кейс |

---

## 7. PreliminaryAssessment (предварительная оценка — совместно с пилотом)

> Рабочая гипотеза на основе оценок пилота. Требует уточнения по обратимости и стоимости ошибки.

| Характеристика | Лучший вариант | Комментарий пилота |
|----------------|----------------|--------------------|
| 1. Время до первого результата | B (контроль) | Систему контроля можно создать и внедрить быстро; для автоматической обработки отчётов не нужна система обучения. |
| 2. Масштабируемость | A (обучение) | Формализованное обучение охватывает больше технологов без роста времени пилота. |
| 3. Качество передачи знаний | A (обучение) | Глубокое и формализованное понимание метода. |
| 4. Риск ошибки применения | B и C (контроль и саморазвитие) | Личный контроль пилота или автоматический контроль дают меньше ошибок, чем переданное через обучение знание. |
| 5. Автономность от пилота | A (обучение) | Обученные технологи работают без пилота; контроль требует вмешательства; саморазвитие завязано на пилота. |
| 6. Стоимость ошибки для бизнеса | Неясно | Требуется дополнительный анализ кейсов ошибок. |
| 7. Обратимость решения | B (контроль) и C (саморазвитие) | Вариант A (обучение) требует больше всего времени на создание, поэтому от него сложнее всего отказаться. |
| 8. Доступность доказательств | Примерно равно | Для всех трёх вариантов данных примерно одинаково. |

---

## 8. Boundaries (границы)

### Остановиться
> Остановиться, когда по каждой характеристике можно поставить осмысленную оценку с указанием источника и неопределённости. Не нужны точные числа — нужна reviewable картина.

### Вернуться
> Вернуться, если появятся новые варианты (например, «обучить одного технолога-лидера», «купить готовое решение»), изменятся критерии или появятся новые данные пилота.

### Следующий шаг после заполнения
> Применить `C.11 Solution -> ChoiceResult` для локального выбора или зафиксировать, что выбор пока не сделан из-за высокой неопределённости.  
>  
> **Обновление:** `ChoiceResult` сформирован — выбран `probe` варианта B (система контроля). См. `2026-07-16-choice-result-training-system.md`.
