## Замечания по лексике

1. **ГРР** — в транскрипте используется как синоним DRR. Нужно либо унифицировать, либо явно указать, что ГРР — локальное название.
2. **ЛПФ / ДПФ** — личные/доменные фреймворки, не путать с FPF.
3. **Кампейн / workstream** — не FPF-термины, а локальные обозначения спикера.
4. **Констрактор** — при переводе в FPF лучше использовать «трансформер».

## Рекомендации

- Добавить glossary в начало семинарных материалов.
- Явно помечать локальные термины (local term) в рендерингах.
- Унифицировать ГРР → DRR или дать отдельное определение.

## Boundary

E.10 распознаёт и закрывает локальные лексические проблемы или выбирает применимую строку. Для восстановления онтологической точности — E.10.ARCH.
