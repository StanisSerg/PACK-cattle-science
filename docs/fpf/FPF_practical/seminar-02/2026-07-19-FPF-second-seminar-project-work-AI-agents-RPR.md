# Relational Precision Restoration (A.6.P / A.6.A) для семинара FPF-02

## Примеры восстановления kind-explicit relations

| Размытая формулировка (из транскрипта) | Восстановленное kind-explicit relation | Пояснение |
|----------------------------------------|----------------------------------------|-----------|
| "система действует" | system::enacts-as-whole | Система действует как организованное целое |
| "результат — это хвост отношения" | result::stable-name-in-relation | Результат — имя в отношении, не объект |
| "агенты работают" | agent::performs-role-as-executor / agent::performs-role-as-reviewer | Работа агента — это роль, а не самоагентность |
| "чашка в семинаре" | cup::assigned-role-as(teaching-aid) | Роль, не тип |
| "код не действует сам" | code::episteme-describes(method-of, computer) | Код — эпистема, описывающая метод работы компьютера |
| "программа работает" | running-program::physical-system-executes(method) | Действующая программа — физическая система |
| "характеристика важна" | characteristic::architecturally-significant-for(decision) | Характеристика значима для конкретного решения |
| "печень хочет расти" | liver-cells::local-optimization vs organism::global-optimization | Конфликт уровней |
| "флот появляется" | aircraft::parts-compose → fleet::emergent-whole | Метахолонный переход |
| "слайдомент создан агентами" | slide-ument::co-produced-by(human-architect, agents) | Совместное производство, а не автоматическое |

## CC-A.6.P-1: Explicit relation family and explicit sense

В каждой строке выше relation family и sense сделаны явными. Это устраняет overload, при котором одно и то же слово («система», «работает», «важно») может относиться к разным kind.
