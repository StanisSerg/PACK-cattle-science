# Кейс: консультант по кормлению и оценка рациона через 2 недели

> Классический пример смешения WorkPlan, Work и Description в практике cattle science. Разбор по FPF.

---

## Сценарий

1. Консультант по кормлению приезжает на ферму.
2. Оценивает ситуацию и составляет **рекомендуемый рацион**.
3. Уезжает.
4. Через 2 недели возвращается.
5. Смотрит на свой **рекомендованный рацион** и на **продуктивность**.
6. Делает вывод: рацион работает или не работает.

---

## FPF-разбор

### 1. System (система)

- Стадо коров на ферме.
- Реальный корм в кормушке.
- Условия содержания, доение, климат, состояние здоровья.

### 2. Epistemes (эпистемы), присутствующие в сценарии

| Элемент | Вид эпистемы | Пояснение |
|---------|--------------|-----------|
| Первичная оценка консультанта | Description + Pre-articulation cue | «Вот такая ситуация на ферме» — частично оформлено, частично интуиция. |
| Рекомендуемый рацион | **WorkPlan** + **MethodDescription** | «Нужно сделать вот так». Это план, а не факт. |
| Продуктивность через 2 недели | **Measurement** / **Description** | «Удой изменился на X%». |
| Вывод «работает / не работает» | **Description** / **Dynamics-based claim** | Утверждение о причинно-следственной связи. |

### 3. Work (работа) — ключевое пропущенное звено

Консультант строит цепочку:

> **WorkPlan** → через 2 недели → **Outcome**

Но в FPF корректная цепочка:

> **WorkPlan** → **Work** (что реально сделали) → **Outcome** → **Evidence Graph**

**Work** — это фактически выполненные действия: что реально кормили, в каких количествах, когда, кем, с какими отклонениями.

---

## Чего не хватает для корректного вывода

| Что нужно | Почему важно |
|-----------|--------------|
| **Work record** — что реально кормили | Рекомендация ≠ действие. Фермер мог отступить от рациона по цене, наличию корма или привычке. |
| **Measurement фактического рациона** | Состав корма в кормушке может отличаться от расчётного. |
| **Confounders** — другие эпистемы | За 2 недели могли измениться: погода, доение, здоровье, стресс, лабораторная ошибка. |
| **Language-state позиции** | 2 недели — часто слишком рано; conclusion должна быть максимум **candidate**, не **settled**. |
| **Evidence Graph** | От claim «рацион работает» должна быть явная цепочка: plan → work → measurement → outcome. |

---

## Почему вывод «работает / не работает» слабый

### Если продуктивность выросла

Возможные причины:

1. Рацион действительно работает.
2. Рацион не выполняли, но улучшилось доение по другой причине.
3. Прошла жара, и коровы восстановились.
4. Фермер параллельно ввёл добавку.
5. Измерение продуктивности было неточным.

Без Work record и контроля confounders невозможно отделить вариант 1 от вариантов 2–5.

### Если продуктивность упала

Аналогично: виноват ли рацион, или сработал другой фактор?

---

## Как должна выглядеть корректная FPF-последовательность

### Этап 1. Первый визит

| Элемент | FPF-класс |
|---------|-----------|
| Стадо, рацион, продуктивность | EntityOfConcern / System |
| Текущее состояние | Description |
| Метод оценки и расчёта | MethodDescription |
| Рекомендуемый рацион + план внедрения | WorkPlan |

### Этап 2. Промежуточный период

| Элемент | FPF-класс |
|---------|-----------|
| Что фермер реально кормил, когда и сколько | Work record |
| Фактический состав рациона | Measurement |
| Погода, здоровье, доение, стрессоры | Description / Measurement (confounders) |

### Этап 3. Второй визит

| Элемент | FPF-класс |
|---------|-----------|
| Продуктивность, жирность, белок, BHB | Measurement |
| Связь Work + confounders → outcome | Evidence Graph |
| Вывод | Description / Dynamics-based claim со scope и confidence level |

---

## Пример корректного FPF-вывода

Вместо:

> «Рацион работает / не работает.»

Должно быть:

> «При фактическом выполнении рациона на 85% (Work record) и отсутствии значимых стрессоров (confounders) продуктивность выросла на 4% за 2 недели (Measurement). Пока это **candidate-stage claim**: возможно, эффект связан с рационом, но требуется ещё 2–3 недели и контроль других факторов для перехода к settled.»

---

## Нарушенные FPF-принципы

| Принцип | Как нарушен |
|---------|-------------|
| **A.7 Strict Distinction** | WorkPlan принят за Work. |
| **A.10 Evidence Graph** | Нет цепочки plan → work → measurement → outcome. |
| **A.1.1 Bounded Context** | Рекомендация сделана в одних условиях; через 2 недели контекст мог измениться. |
| **A.6.B L/A/D/E** | Вывод смешивает Work-Effect, Law и Admissibility. |
| **A.16 Language-state** | Вывод сформулирован как settled, хотя это максимум candidate. |

---

## Мораль кейса

> **Нельзя судить о рационе по рекомендации. Нужно судить о рационе по тому, что реально кормили, и по тому, какие ещё факторы действовали.**

Консультант не ошибается в профессионализме. Он ошибается в **форме знания**: принимает собственный WorkPlan за достаточное основание для causal claim.
