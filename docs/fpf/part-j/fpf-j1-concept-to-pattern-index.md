# J.1 — Concept-to-Pattern Index: указатель от концепции к паттерну

> **Цель capture:** объяснить, как использовать указатель от концепции к паттерну для быстрого перехода от разговорной идеи к формальному ID паттерна и разделу FPF.

---

## 1. Зачем это читать

В разговорной практике специалисты говорят о «границе», «риске», «доказательстве», «норме». Эти слова звучат понятно, но в FPF за каждым из них стоит конкретный паттерн с точным ID. Concept-to-Pattern Index помогает не искать паттерн вслепую, а сразу переходить от слова к управляющему разделу.

> **FPF-тезис:** *«Идея без ID паттерна — это ещё не рабочий термин.»*

**Фермерский пример:**

> Ветеринар говорит: «Нужно оценить границу риска кетоза». Без указателя команда может интерпретировать «границу» как пространственную, временную или область действия. Concept-to-Pattern Index направляет к A.6.C для области действия утверждения и к C.2 для риска.

---

## 2. История одной ошибки

В хозяйстве разработали новый протокол оценки «boundary» переходного периода. Команда обратилась к паттерну A.6.B «Spatial Boundary», потому что слово boundary ассоциировалось с физическим пространством. На самом деле речь шла о временном интервале вокруг отёла, который управляется C.26. Протокол пришлось переписывать.

---

## 3. Concept-to-Pattern Index — полное описание

### 3.1 Концепция как вход

**Определение.** Концепция — это разговорное или предметное понятие, которое практик использует для описания задачи до формализации в FPF.

**Пояснение.** Концепции не являются терминами FPF. Они служат точкой входа. Задача указателя — преобразовать концепцию в один или несколько формальных паттернов.

**Пример из животноводства.** Концепция «кетоз» может вести к C.2 (risk class), C.3 (clinical diagnosis), C.5 (economic loss) или C.21 (metabolic state) в зависимости от контекста.

**Ключевой признак.** Если одна концепция ведёт к нескольким паттернам, нужен дополнительный контекст для выбора.

### 3.2 Паттерн-цель

**Определение.** Паттерн-цель — это формальный ID паттерна, который управляет выбранной концепцией в конкретном контексте.

**Пояснение.** Одна концепция может иметь несколько целевых паттернов. Указатель показывает все возможные варианты и указывает, какой дополнительный вопрос поможет выбрать правильный.

**Пример из животноводства.** Концепция «BHB» → A.10 (происхождение наблюдения), C.2 (классификация риска), K.1 (измерительная терминология). Для выбора нужно знать, о каком аспекте BHB идёт речь.

**Ключевой признак.** Хороший указатель не только даёт ID, но и подсказывает, какой контекст определяет выбор.

### 3.3 Проблема многозначности

**Определение.** Многозначность концепции — это ситуация, когда одно слово соответствует нескольким паттернам, и выбор зависит от Problem frame.

**Пояснение.** Concept-to-Pattern Index не устраняет многозначность автоматически. Он показывает её и предлагает набор уточняющих вопросов.

**Пример из животноводства.** Слово «norm» может означать среднее по стаду (C.21), целевой интервал (C.2) или клинический порог (C.3). Указатель предлагает спросить: «Это описание, цель или порог принятия решения?»

**Ключевой признак.** Если указатель выдаёт несколько паттернов, значит концепция многозначна и требует уточнения.

---

## 4. Почему смешивать / игнорировать — значит рисковать

Рассмотрим типичное смешанное утверждение:

> *«Работаем с границей стада.»*

**Разложение по J.1:**

| Часть утверждения | Что это в FPF | Почему важно разделять |
|---|---|---|
| «граница» | A.6.B / A.6.C / C.26 | Пространство, область или интервал |
| «стада» | A.1.1 / A.6.C | Индивидуумы или учётная единица |
| Неуказанный контекст | — | Риск выбора неправильного паттерна |

---

## 5. Как это выглядит на ферме: правильное применение

**Было (смешанное / нечёткое):**

> В протоколе написано: «Определить норму BHB». Консультант не знал, что «норма» может быть тремя разными паттернами, и выбрал первый попавшийся — среднее по стаду. В результате целевой порог для принятия решения остался не определён.

**Стало (разложенное / ясное):**

> В протоколе указано: «Если под нормой понимается среднее по стаду → C.21; целевой интервал → C.2; клинический порог → C.3. Выбор фиксируется в заголовке отчёта.»

---

## 6. Практическое применение: с чего начать

1. Запишите ключевые слова, которыми описывается задача.
2. Для каждого слова найдите соответствующие паттерны в Concept-to-Pattern Index.
3. Если одно слово даёт несколько паттернов, определите Problem frame.
4. Выберите паттерн и зафиксируйте его ID в документе.
5. При изменении задачи повторите поиск, так как контекст мог смениться.

---

## 7. Проверь себя

| Вопрос | Если ответ «да» — проблема |
|---|---|
| Вы используете разговорные термины без проверки в указателе? | Риск неправильной маршрутизации |
| Один термин ведёт к нескольким паттернам, но выбор не обоснован? | Контекст не зафиксирован |
| В документе нет ID паттерна, только слово? | Невозможно проверить корректность |
| Указатель не обновляется при появлении новых паттернов? | Поиск может давать устаревшие результаты |

---

## 9. Что запомнить

1. Concept-to-Pattern Index переводит разговорные концепции в формальные ID паттернов.
2. Одна концепция может соответствовать нескольким паттернам.
3. Для выбора паттерна нужен Problem frame и контекст.
4. Указатель не устраняет многозначность, но делает её явной.
5. Всегда фиксируйте ID выбранного паттерна в документе.
