# J.2 — Pattern-to-Example Index: указатель от паттерна к примеру

> **Цель capture:** объяснить, как таблица указателя от паттерна к примеру помогает найти архетипический пример или grounding vignette для каждого паттерна и убедиться, что пример действительно иллюстрирует нужный паттерн.

---

## 1. Зачем это читать

Теория паттернов лучше усваивается через конкретные ситуации. Однако в большом документе примеры разбросаны по разделам, и не всегда очевидно, какой vignette соответствует выбранному паттерну. Pattern-to-Example Index собирает все архетипические примеры в одном месте и связывает их с ID паттернов, чтобы читатель мог быстро найти иллюстрацию.

> **FPF-тезис:** *«Паттерн без примера остаётся абстракцией; пример без паттерна — анекдотом.»*

**Фермерский пример:**

> Зоотехник изучает паттерн A.10 «Evidence-Provenance DAG» и хочет понять, как он применяется к BHB-скринингу. Без указателя ему пришлось бы читать все capture в поисках подходящего примера. С указателем он сразу находит vignette «Происхождение порога BHB 1.2».

---

## 2. История одной ошибки

В учебном материале для фермеров приводился пример с кетозом, но без указания, какой паттерн он иллюстрирует. Читатели интерпретировали его как пример к C.2 (классификация риска), хотя автор имел в виду A.10 (происхождение знания). В результате команда обсуждала порог, а не доказательную базу.

---

## 3. Pattern-to-Example Index — полное описание

### 3.1 Grounding vignette

**Определение.** Grounding vignette — это короткий описательный сценарий, который показывает паттерн в действии в реальном или типизированном контексте.

**Пояснение.** Vignette не заменяет определение паттерна. Она делает его осязаемым. Хорошая vignette содержит все элементы паттерна, но в повседневной форме.

**Пример из животноводства.** Для C.2 grounding vignette: «Корова №1847 на 7-й день лактации имеет BHB 1.4 ммоль/л по фотометрическому методу. Согласно порогу ≥ 1.2, она отнесена к risk class «субклинический кетоз».»

**Ключевой признак.** Если после чтения vignette можно выделить элементы паттерна, пример подходит.

### 3.2 Архетипический пример

**Определение.** Архетипический пример — это стандартный, часто повторяющийся сценарий, который используется для объяснения паттерна разным аудиториям.

**Пояснение.** Архетипы выбираются так, чтобы быть узнаваемыми для целевой аудитории. В cattle-science это часто сценарии с BHB, кетозом, metabolic transition, ветеринарными протоколами.

**Пример из животноводства.** Архетип для K.1: «В старом SOP написано «axis BHB». В новом протоколе это должно быть «Characteristic BHB».»

**Ключевой признак.** Архетипический пример должен быть простым, но содержать ключевое различие, которое демонстрирует паттерн.

### 3.3 Связь примера с паттерном

**Определение.** Связь примера с паттерном — это явное указание, какие элементы паттерна присутствуют в примере и как они проявляются.

**Пояснение.** Недостаточно сказать «это пример C.2». Нужно показать, где в примере observation, threshold, risk class, classification relation. Это позволяет проверить корректность применения.

**Пример из животноводства.** Для vignette с BHB 1.4:

| Элемент C.2 | В примере |
|---|---|
| Observation | BHB 1.4 ммоль/л |
| Threshold | ≥ 1.2 ммоль/л |
| Risk class | субклинический кетоз |
| Relation | `classifiedBy` |

**Ключевой признак.** Каждый пример в указателе должен содержать разбор по элементам паттерна.

---

## 4. Почему смешивать / игнорировать — значит рисковать

Рассмотрим типичное смешанное утверждение:

> *«Вот пример с кетозом — он про риск.»*

**Разложение по J.2:**

| Часть утверждения | Что это в FPF | Почему важно разделять |
|---|---|---|
| «пример с кетозом» | vignette | Содержит много элементов |
| «про риск» | C.2 | Это один из возможных паттернов |
| Неуказанные элементы | A.10 / C.3 / C.5 | Пример может иллюстрировать и другие паттерны |

---

## 5. Как это выглядит на ферме: правильное применение

**Было (смешанное / нечёткое):**

> В обучающем материале был пример: «Корова с BHB 1.4 получила пропиленгликоль». Не было понятно, иллюстрирует ли он классификацию риска, клиническое решение или экономический эффект.

**Стало (разложенное / ясное):**

> Каждый пример снабжён таблицей: «Паттерн: C.2; элементы: observation BHB 1.4, threshold ≥ 1.2, risk class; если нужен пример клинического решения — см. C.3 vignette №12.»

---

## 6. Практическое применение: с чего начать

1. При изучении нового паттерна откройте Pattern-to-Example Index.
2. Найдите vignette, связанную с вашим паттерном.
3. Проверьте, что все элементы паттерна присутствуют в примере.
4. Если пример неясен, найдите альтернативный в том же указателе.
5. При создании собственных материалов связывайте примеры с ID паттернов.

---

## 7. Проверь себя

| Вопрос | Если ответ «да» — проблема |
|---|---|
| Пример приводится без указания паттерна? | Читатель может интерпретировать его иначе |
| Вы не можете выделить элементы паттерна в примере? | Пример плохо иллюстрирует паттерн |
| Один пример используется для разных паттернов без пояснения? | Возможно смешение |
| В указателе отсутствуют примеры для важных паттернов? | Обучение будет абстрактным |

## 8. Связь с другими паттернами

| Паттерн | Связь |
|---|---|
| J.1 | Concept-to-Pattern Index помогает найти паттерн, для которого нужен пример |
| H.2 | Каталог паттернов показывает статус паттерна, к которому ведёт пример |
| A.10 | Evidence-Provenance DAG часто иллюстрируется через BHB-vignette |
| C.2 | Классификация риска — один из наиболее частых источников примеров |

---

## 9. Что запомнить

1. Pattern-to-Example Index связывает каждый паттерн с архетипическим примером.
2. Пример не заменяет определение, но делает паттерн понятным.
3. Каждый пример должен содержать разбор по элементам паттерна.
4. Один сценарий может иллюстрировать несколько паттернов — важно указывать, какой именно.
5. При создании учебных материалов всегда связывайте пример с ID паттерна.
