# Decisions / Decision Layer

> Из кейса рождается решение.  
> Вопрос: «что именно здесь было решением?»

## Что такое DL?

**Decision Layer** — промежуточный слой между сырым кейсом и формализованным правилом.

**Задача DL:** Вытащить из кейса:
- условие
- какое правило сработало
- какой вердикт был принят

## Структура DL

```
IF [условия из кейса]
THEN [что было решено]
BECAUSE [почему это сработало]
```

**Важно:** DL — ещё не универсальное правило. Это решение конкретного случая.

## Переход к правилу

**Вопрос:** «это единичный случай или закономерность?»

| Ответ | Действие |
|-------|----------|
| Единичный | Оставить как DL (документация опыта) |
| Закономерность | Создать RULE в pack/rules/ |
