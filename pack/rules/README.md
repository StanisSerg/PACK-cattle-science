# Rules / Правила Pack

> Из DL рождается правило.  
> Формат: IF → THEN → BECAUSE → LIMITS

## Что такое правило?

**Правило** — формализованное знание системы, готовое к применению и автоматизации.

**Отличие от DL:**
- DL — решение конкретного случая
- RULE — универсальный алгоритм

## Структура правила

```yaml
IF:    [измеримые условия]
THEN:  [что делать / не делать]
BECAUSE: [механизм — почему работает]
LIMITS: [где работает / не работает]
```

### IF
Чёткие, измеримые условия:
- BHB > 1.2 mmol/L
- DMI < 80% от нормы
- Потеря BCS > 0.5

### THEN
Конкретные действия:
- НЕ применять X
- Приоритет: коррекция системы
- Мониторинг: каждые N дней

### BECAUSE
Объяснение механизма:
- Почему это работает
- Какие процессы происходят
- Почему другие варианты не подходят

### LIMITS
Границы применимости:
- ✅ Где работает
- ❌ Где НЕ работает
- ⚠️ Требует осторожности

## Пример полного цикла

```
CASE-001: BHB 1.6 → стандарт → нет эффекта → система → BHB 0.4
    ↓
DL-001: IF BHB > 1.2 + низкий DMI → не X → коррекция системы
    ↓
RULE-001: Ketosis-Threshold-Invalidation
    IF: BHB > 1.2 AND DMI < 80%
    THEN: не применять гепатопротекторы как основное
    BECAUSE: метаболический дефицит → нет субстратов
    LIMITS: ранний постпартум, нет клиники
```

## Именование файлов

```
RULE-NNN-{краткое-название}.md

Примеры:
- RULE-001-ketosis-threshold.md
- RULE-002-mastitis-early-protocol.md
```

## Валидация

**Confidence levels:**
- `high` — много валидированных кейсов
- `medium` — несколько кейсов, требует проверки
- `low` — один кейс или теоретическое обоснование

**Требования для повышения confidence:**
- [ ] Минимум 3 подтверждающих кейса
- [ ] Нет противопоказаний
- [ ] Документированы LIMITS

## Автоматизация

Правила с `confidence: high` → кандидаты для:
- Триггеров в системе мониторинга
- Рекомендаций AI-агента
- Автоматических алертов
