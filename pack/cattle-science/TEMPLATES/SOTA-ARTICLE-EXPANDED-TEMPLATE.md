# SoTA Article Expanded Format: Руководство по написанию полноценных State-of-the-Art для статей

> **Проблема:** Стандартные SoTA для статей содержат только summary + key findings.  
> **Решение:** Добавить механистический контекст, критический анализ методологии, сравнение с другими исследованиями и практическую трансляцию.  
> **FPF-статус:** Полностью compliant. ConservativeRetextualization (A.6.3) разрешает подробный пересказ, если сохраняется describedEntity и есть reopen trigger (DOI, страницы).

---

## 1. Что такое "расширенный" SoTA для статьи

| Аспект | "Сухой" SoTA (старый) | "Расширенный" SoTA (новый) |
|--------|----------------------|---------------------------|
| **Цель** | Быстро найти выводы | Понять, почему результаты именно такие, и применимы ли они |
| **Читатель** | Специалист, помнящий контекст | Специалисты, преподаватели, аспиранты, консультанты |
| **Тон** | Справочный, минималистичный | Академический, формальный, без аналогий и метафор |
| **Структура** | YAML → Утверждения → Результаты | YAML → Гипотеза → Дизайн → Механизмы → Результаты → Трансляция |
| **Объём** | 300–500 строк | 600–1000+ строк |
| **Время изучения** | 15–20 мин | 45–60 мин |

**Ключевое отличие:** Расширенный SoTA не добавляет новых claims. Он *раскрывает* существующие claims из статьи, добавляя механистический, методологический и клинический контекст.

## 2. Принцип "Статья ≠ Глава книги"

> **Правило:** Peer-reviewed статья имеет другую архитектуру, чем глава книги (NASEM, NRC). Шаблон адаптирует структуру Expanded к логике научной статьи.

**Структура статьи → Структура SoTA:**

```
Статья:                        SoTA:
Abstract                       → Аннотация + Key Claims
Introduction                   → 3.2. Контекст + 3.3. Гипотеза
Materials & Methods            → 4. МЕТОДОЛОГИЯ (дизайн, выборка, анализ)
Results (Figure/Table)         → 5. РЕЗУЛЬТАТЫ (по каждому медиа)
Discussion                     → 6. ИНТЕРПРЕТАЦИЯ + 7. КРИТИЧЕСКИЙ АНАЛИЗ
Conclusions                    → 8. ВЫВОДЫ
References                     → 11. ИСТОЧНИКИ
```

**Anti-pattern:** Просто копировать Abstract → Introduction → Results → Discussion в SoTA. Это не добавляет ценности.

## 3. Принципы написания описательных разделов

### 3.1. Правило механистического обоснования — каждый результат должен иметь механизм

**Плохо:**
```markdown
EGTA-третированные коровы имели более высокий iCa постпартум.
```

**Хорошо:**
```markdown
**Механизм:** EGTA связывает ионизированный Ca в плазме, вызывая транзиторную гипокальциемию. 
Это активирует отрицательную обратную связь через паращитовидный гормон (PTH) и 1,25-(OH)₂-D₃ 
(Frizzarini et al., 2026, p. 4605). Постпартум, когда спрос на Ca резко возрастает 
(30–50 г/сут в молоке), предварительно активированные механизмы резорбции кости и кишечной 
абсорбции обеспечивают более быстрое восстановление iCa.
```

### 3.2. Правило "Модель/Эксперимент ≠ Реальность" (FPF A.7 Strict Distinction)

**Плохо:**
```markdown
EGTA предотвращает субклиническую гипокальциемию.
```

**Хорошо:**
```markdown
**Модель Frizzarini et al. (2026)** демонстрирует, что индуцированная препартум субклиническая 
гипокальциемия (iCa 0,7–1,0 мМ) в условиях контролируемой инфузии EGTA приводит к более высоким 
концентрациям iCa постпартум у многоплодных Holstein (p. 4608). **Важно:** Это экспериментальная 
модель с внутривенной инфузией в исследовательских условиях. Реальная применимость к пищевой 
стратегии (низкий Ca препартум) требует отдельной валидации: форма Ca, скорость абсорбции, 
рубцевая биодоступность и индивидуальная вариабельность не контролировались в данном исследовании.
```

### 3.3. Правило иерархической структуры

Структура описательного раздела должна следовать логике от фундаментального к прикладному:

```
1. Физиологический/биологический контекст (зачем это важно)
2. Патофизиология нарушений (что происходит при дисбалансе)
3. Механизмы гомеостаза (гормоны, органы, транспорт)
4. Экспериментальный дизайн (как это изучали)
5. Ключевые результаты с числовыми значениями
6. Интерпретация и импликации для практики
7. Ограничения и пограничные сценарии
```

### 3.4. Правило "Evidence anchors для описаний" (FPF A.10)

Даже описательный текст должен быть привязан к источнику:

**Плохо:**
```markdown
PTH стимулирует резорбцию кости.
```

**Хорошо:**
```markdown
PTH стимулирует резорбцию кости через активацию остеокластов 
(Frizzarini et al., 2026, p. 4605; Potts, 2005; Goff, 2008).
```

### 3.5. Правило формального академического тона

**Запрещено:**
- Аналогии из бытовой сферы
- Разговорные вопросы в заголовках
- Упрощения типа "просто", "всего лишь"
- Эмоциональные оценки без метрик
- Метафоры

**Разрешено:**
- Терминологически точные формулировки
- Количественные характеристики
- Структурированные сравнения в табличной форме
- Ссылки на первоисточники с номерами страниц

### 3.6. Правило Anti-Hallucination

**Золотое правило:** Если числа, P-значения, доверительные интервалы или выборки не указаны в статье — их нельзя придумывать.

**Метки происхождения каждого блока:**

| Метка | Значение | Пример |
|-------|----------|--------|
| `(Frizzarini et al., 2026, p. XX)` | Прямая цитата/пересказ | `iCa = 1,12 ± 0,02 мМ (Frizzarini et al., 2026, p. 4608)` |
| `[вне статьи: Author, Year]` | Дополнение из внешней литературы | `[вне статьи: Goff, 2008]` |
| `[интерполяция]` | Логический вывод | `[интерполяция: при n = 15, мощность 80 %]` |
| `[guess]` | Практическая догадка | `[guess: применимость к Jersey может отличаться]` |

**Правило:** Если блок не имеет ни одной метки — он считается галлюцинацией и подлежит удалению.

### 3.7. Оценка уверенности в Key Claims

**Запрещено:**
```markdown
**Уверенность:** высокая.
```

**Обязательно:**
```markdown
**Уверенность:** 0,85 (RCT, n = 30, α = 0,05, мощность 80 %; воспроизведено в 2 независимых исследованиях).
**Уверенность:** 0,62 (одно исследование, n = 15/группа, ограничение: только multiparous Holstein).
```

**Формат:** число на шкале 0,0–1,0 + обоснование (тип evidence, объём выборки, метрики точности, ограничения).

## 5. Чек-лист FPF для расширенных разделов

| Паттерн | Требование | Как проверить |
|---------|-----------|---------------|
| **A.7 Strict Distinction** | Каждое описание — о *модели/эксперименте*, не о реальности | Поискать "корова поглощает" → заменить на "в модели Frizzarini корова поглощает" |
| **A.6.3 ConservativeRetextualization** | Нет новых claims без маркировки | Проверить, что каждое утверждение можно найти в статье или помечено [вне статьи] |
| **A.6.3.CSC** | Reopen trigger: DOI + страница | Каждый блок ссылается на (Author, Year, p. XXX) или DOI |
| **A.10 Evidence Graph** | Evidence anchors в описаниях | "PTH активирует остеокластов (Frizzarini et al., 2026, p. 4605; Goff, 2008)" |
| **A.6.Q Quality Term Precision** | Нет субъективных оценок без метрик | "Высокая абсорбция" → "AC = 0,92". "Уверенность: высокая" → "0,85 (n = 30, α = 0,05)" |
| **A.6.P Relation Precision** | Точные отношения | "Ca важен для костей" → "Ca составляет 98 % скелетной массы" |
| **A.15 WorkPlan** | WorkPlan включает шаги | Шаг "Добавить механистический контекст" в WorkPlan |
| **Tone check** | Формальный академический тон | Отсутствие аналогий, метафор, разговорных вопросов |
| **Anti-Hallucination** | Нет придуманных P-значений или выборок | Проверка: каждое число есть в статье |
| **Anti-Hallucination** | Каждый блок имеет метку происхождения | (Author, Year, p. XX) или [вне статьи] или [интерполяция] или [guess] |

## 6. Anti-patterns

### AP-1: "Учебник биологии" без связи со статьёй
**Плохо:** Пять абзацев о биохимии Ca²⁺/кальмодулина без упоминания, как это используется в статье.
**Правило:** Каждый физиологический факт должен вести к объяснению результата.

### AP-2: Смешение эксперимента и личного мнения
**Плохо:** "Я считаю, что EGTA лучше диетической манипуляции."
**Правило:** Мнение должно быть явно помечено как [guess] или [интерполяция].

### AP-3: Отсутствие reopen trigger
**Плохо:** "PTH стимулирует резорбцию кости." (без ссылки)
**Правило:** Каждый механизм должен иметь ссылку: (Frizzarini et al., 2026, p. 4605)

### AP-4: Перегрузка терминами
**Плохо:** "1,25-(OH)₂-D₃ связывается с VDR, формируя гетеродимер с RXR..."
**Правило:** Сначала функциональное обобщение, затем детализация.

### AP-5: Разговорный тон и аналогии
**Плохо:** "Корова — не просто потребитель корма, а хозяин симбиотической экосистемы."
**Правило:** Академический регистр, терминологическая точность.

## 7. Итог: баланс "описание vs результат"

| Компонент | Доля | Что включает |
|-----------|------|--------------|
| **Описательный текст** | 50–60 % | Механизмы, контекст, обоснование, сравнения |
| **Результаты с числами** | 20–25 % | P-значения, LSM ± SEM, CI |
| **Таблицы и скриншоты** | 10–15 % | Оригинальные таблицы из статьи (целые страницы) |
| **Практическая трансляция** | 10–15 % | Алгоритмы, типичные ошибки, пограничные сценарии |
