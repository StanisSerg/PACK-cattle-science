# Рабочие продукты (Work Products)

> **Назначение:** Результаты применения методов к конкретным ситуациям  
> **Аналогия:** Метод = рецепт, WP = приготовленное блюдо  
> **Для кого:** Консультанты, ветеринары, менеджеры хозяйств

## Принципы создания WP

1. **WP ≠ Метод**
   - Метод — универсальный протокол
   - WP — применение протокола к конкретной ситуации с данными

2. **WP содержит:**
   - Конкретные данные (BHB, фенотипы, ID коров)
   - Конкретные рекомендации для данного хозяйства
   - Экономические расчёты
   - План действий с ответственными и сроками

3. **WP НЕ содержит:**
   - Общие описания методов (→ ссылка на METHOD)
   - Общие пороги без применения (→ ссылка на METHOD)
   - Теоретические обоснования (→ ссылка на SoTA)

---

## Связь Method → Work Product

```
CS.METHOD.001 (Протокол кетоза)
      ↓ Применение к стаду XYZ
CS.WP.001-[2026]-[001] (Отчёт для хозяйства XYZ)
      ↓ Содержит:
      - Данные скрининга стада XYZ
      - Выявленные проблемы
      - Конкретные рекомендации
      - Экономический расчёт
      - План контроля
```

---

## Нумерация WP

**Формат:** `CS.WP.XXX-[YYYY]-[NNN]`

| Компонент | Описание |
|-----------|----------|
| XXX | Тип WP (001 = метаболический статус) |
| YYYY | Год составления |
| NNN | Порядковый номер в году |

**Пример:** `CS.WP.001-2026-042` = 42-й отчёт о метаболическом статусе в 2026 году
