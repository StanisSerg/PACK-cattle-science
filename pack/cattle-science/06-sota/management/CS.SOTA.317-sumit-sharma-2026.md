# 2. РЕЗЮМЕ (Abstract)

## 2.1. Перевод Abstract

Точное молочное животноводство (precision dairy farming, PDF) быстро трансформирует глобальный молочный сектор через применение технологий, основанных на данных. Этот обзор исследует текущие знания и новые идеи в 5 ключевых областях: (1) экономическая ценность данных; (2) интеграция ИИ, устойчивости и инноваций; (3) драйверы и барьеры внедрения технологий; (4) устойчивое управление данными; (5) межсекторные уроки здравоохранения. Представлены рекомендации для участников отрасли.

# 3. ВВЕДЕНИЕ (Introduction)

## 3.1. Полный текст введения [перевод]

Точное молочное животноводство (PDF) использует передовые технологии мониторинга в реальном времени: доильные роботы, сенсоры, системы компьютерного зрения на базе камер. Эти технологии собирают структурированные (таблицы) и неструктурированные (ветеринарные отчёты, данные сенсоров, рыночные индикаторы) данные для поддержки принятия решений.

Ценность данных обусловлена их свойствами: неисчерпаемость, лёгкость транспортировки, сложность оценки стоимости, усиление ценности при использовании, скоропортящийся характер, убывающая отдача при избытке (Moody and Walsh, 1999). Технологии, генерирующие данные (автоматические доильные системы, системы обнаружения хромоты), производят большие объёмы, требующие интерпретации и своевременных действий.

## 3.2. Ключевые аргументы автора

- Данные — стратегический актив, но их экономическая ценность сложно измерить.
- Интеграция данных через цифровые платформы и облачные системы улучшает принятие решений.
- Отсутствие интероперабельности между системами разных поставщиков — ключевой барьер.
- Устойчивое управление данными требует стандартов, прозрачности и кибербезопасности.
- Уроки здравоохранения (EHR, FAIR) применимы к молочной отрасли.

## 3.3. Литература для сравнения

- **Cabrera, 2018** — инструменты поддержки решений на основе данных улучшают экономическую эффективность фермеров.
- **Cabrera et al., 2020** — симпозиум: принятие решений в реальном времени с использованием больших данных на молочных фермах.
- **Brito et al., 2025** — использование данных сенсоров для улучшения устойчивости селекционных программ.
- **Moody and Walsh, 1999** — свойства данных как экономического актива.

# 4. ОБЗОР ПО 5 ОБЛАСТЯМ

## 4.1. Экономическая ценность данных (p. 3138-3142)

### 4.1.1. Основные положения

Данные приносят экономическую ценность всем участникам цепочки создания стоимости: прямые экономические выгоды, экологическая устойчивость, социальные блага. Примеры:
- **Автоматические доильные системы (AMS):** снижение трудозатрат, улучшение качества молока, улучшение взаимодействия человек-животное (Liseune et al., 2021).
- **Сенсоры хромоты:** улучшение благополучия, снижение случаев хромоты, улучшение гигиены коров (Lavrijsen-Kromwijk et al., 2024).
- **Прогнозирование срока годности молока:** сочетание оlfactory sensing + ML (multilayer perceptron) → точность 85-100% для максимального срока годности (Mamat et al., 2024).

### 4.1.2. Ограничения и вызовы

- Экономическая отдача зависит от состояния фермы до внедрения.
- Выгоды для благополучия животных сложно монетизировать (Cogato et al., 2021).
- Будущие исследования: идентификация ключевых входных данных (здоровье, продуктивность) с наиболее измеримыми экономическими результатами.

### 4.1.3. Dairy 4.0

«Dairy 4.0» — использование робототехники, AI и IoT для повышения устойчивости и эффективности крупномасштабных молочных операций (Sangode, 2024).

## 4.2. AI, устойчивость и инновации (p. 3142-3146)

### 4.2.1. ИИ и машинное обучение

ИИ — общий термин для вычислительных систем, выполняющих задачи, требующие человеческого интеллекта. ML — подмножество, изучающее предсказательные зависимости из примеров.

**Применения:**
- **MIR-спектроскопия:** временная шкала развития анализа молока в Бельгии (Figure 2). MIR используется для предсказания жирных кислот молока, энергетического баланса и других показателей.
- **Геномика + фенотипирование:** интеграция геномных данных с высокочастотными фенотипами из сенсоров. Пример: >5000 коров Holstein, 36 роботизированных доильных станций → поведенческие признаки (среднее время доения, время между доениями, число попыток, процент успешных доений, консистентность предпочтений). Наследуемости: 0,10-0,46 (Bérat et al., 2025).
- **Кормовая эффективность:** мета-анализ Holstein — RFI h² ≈ 0,20, DMI h² ≈ 0,34, энергетический баланс h² ≈ 0,22-0,24 (Jiang et al., 2024).
- **Сенсорные признаки здоровья и благополучия:** входят в генетические оценки.

### 4.2.2. Мультисенсорное слияние и цифровые двойники

Интеграция видео + сенсоров + геномики повышает точность предсказания субклинических состояний (кетоз, хромота) по сравнению с одноисточниковыми моделями (Ferreira et al., 2024; Menezes et al., 2025).

**Цифровые двойники (digital twins):** виртуальные представления коров или стад. Обеспечивают мониторинг в реальном времени и предиктивную аналитику, моделируя состояние коровы при различных сценариях управления (Neethirajan and Kemp, 2021; Hostens, 2024).

### 4.2.3. Будущие направления

- Полностью автоматизированные инструменты поддержки решений (AHM).
- Компьютерное зрение, дроны, edge computing.
- Обработка естественного языка (NLP) для агрегации неструктурированных данных (ветеринарные отчёты, заметки фермеров, социальные медиа).

## 4.3. Барьеры внедрения технологий (p. 3146-3148)

### 4.3.1. Ключевые барьеры

1. **Конфиденциальность и безопасность данных:** фермеры неохотно делятся данными без гарантий защиты (Dagne, 2021).
2. **Непрозрачность владения данными:** EULA компаний часто содержат пункты о владении данными, которые фермеры не читают/не понимают.
3. **Кладбища технологий (technology graveyards):** технологии, не решившие практических задач ферм, стали устаревшими (Bewley, 2024).
4. **Отсутствие интероперабельности:** системы разных поставщиков работают изолированно.
5. **Недостаток цифровых навыков:** нехватка специалистов по обработке данных в сельском хозяйстве.
6. **Нестандартные форматы данных:** PDF, email — неинтерактивные форматы ограничивают анализ (Cabrera, 2024).

### 4.3.2. Решения

- Прозрачные соглашения о совместном использовании данных.
- Вовлечение фермеров в принятие решений.
- Стимулы для обмена данными (как модели Sysco в рознице).
- Стандартизация ID животных: ICAR сертифицирует RFID в соответствии с ISO (Rosati, 2011).

## 4.4. Устойчивое управление данными (p. 3148-3150)

### 4.4.1. Качество данных и стандарты

Высокое качество данных — фундамент PDF. Стандартизированные признаки необходимы для точной записи (Cabrera et al., 2020).

**ADAPT (Ag Data Application Programming Toolkit):** стандартизированный инструмент для единообразной обработки данных в агропродовольственной цепочке (Kemp, 2024).

### 4.4.2. Национальные базы данных

Национальные базы данных играют ключевую роль в управлении высокочастотными данными качества молока. Переход от USDA к CDCB улучшил устойчивость за счёт снижения зависимости от федерального финансирования (Weigel, 2024). Это позволяет развивать новые генетические оценки для кормовой эффективности и выбросов метана.

### 4.4.3. Принципы FAIR

- **Findable (Находимые):** уникальные идентификаторы и богатые метаданные.
- **Accessible (Доступные):** чёткие протоколы получения.
- **Interoperable (Интероперабельные):** общие стандарты для бесшовной интеграции.
- **Reusable (Многоразовые):** лицензирование и информация о происхождении (Wilkinson et al., 2016).

### 4.4.4. Кибербезопасность

Кибербезопасность критична для защиты данных ферм от несанкционированного доступа и кибератак. Permissioned blockchain может поддерживать целостность данных, контроль доступа и аудит (Makarov et al., 2019; Carter et al., 2020).

## 4.5. Межсекторные уроки: молочное производство и здравоохранение (p. 3150-3151)

### 4.5.1. Параллели

Оба сектора пережили значительную трансформацию в сборе, обработке и использовании данных. Интеграция данных в здравоохранении улучшила уход за пациентами; в молочном производстве — управление стадом, благополучие животных и продуктивность.

### 4.5.2. Применимые уроки

- **Электронные медицинские карты (EHR):** аналог — электронные ветеринарные карты и базы данных стад.
- **Предиктивная аналитика:** раннее выявление заболеваний в обоих секторах.
- **Управление данными:** стандарты, интероперабельность, управление доступом.
- **Соответствие нормативам:** прослеживаемость продукции.

# 5. РЕКОМЕНДАЦИИ (Recommendations)

## 5.1. Рекомендации автора (p. 3151-3152)

1. **Инвестиции в инфраструктуру данных:** облачные системы, API, стандарты интероперабельности.
2. **Образование и цифровые навыки:** интеграция computer science в ветеринарное и сельскохозяйственное образование.
3. **Внедрение прорывных технологий:** AI, цифровые двойники, IoT, дроны, edge computing, blockchain.
4. **Максимизация ценности данных:** real-time аналитика, предиктивное обслуживание, количественная оценка финансового эффекта.
5. **Устойчивое управление данными:** FAIR, чёткое владение, кибербезопасность, национальные базы данных.
6. **Междисциплинарное сотрудничество:** объединение усилий исследователей, индустрии, фермеров, политиков.

# 7. FAQ

**Q1: Что такое Dairy 4.0?**
A: Концепция интеграции робототехники, AI и IoT для повышения устойчивости и эффективности крупномасштабного молочного производства.

**Q2: Какие барьеры мешают внедрению технологий на фермах?**
A: (1) Конфиденциальность и безопасность данных; (2) непрозрачность владения данными; (3) отсутствие интероперабельности между системами; (4) недостаток цифровых навыков; (5) «кладбища технологий» — решения, не отвечающие реальным потребностям.

**Q3: Каковы наследуемости новых сенсорных признаков?**
A: Поведенческие признаки (время доения, консистентность предпочтений): h² = 0,10-0,46. Кормовая эффективность (RFI): h² ≈ 0,20.

**Q4: Что такое цифровые двойники в молочном производстве?**
A: Виртуальные модели коров или стад, позволяющие моделировать сценарии управления и прогнозировать состояние в реальном времени.

**Q5: Какие принципы FAIR применимы к данным в животноводстве?**
A: Findable (находимые), Accessible (доступные), Interoperable (интероперабельные), Reusable (многоразовые).

**Q6: Как применить эти результаты в России?**
A: (1) Развивать национальные базы данных (аналог CDCB); (2) внедрять стандарты ICAR для идентификации; (3) создавать образовательные программы по data science для аграриев; (4) стимулировать интероперабельность отечественных систем мониторинга.
