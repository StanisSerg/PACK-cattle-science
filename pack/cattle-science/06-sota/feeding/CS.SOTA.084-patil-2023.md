# CS.SOTA.084: Patil et al. (2023) — Стохастическое программирование для наименее затратного формулирования кормов

> **Уровень:** Продвинутый (M2/S1)  
> **Формат:** Simulation/Modeling  
> **Время изучения:** 50-60 минут  
> **Применение:** Оптимизация рационов, управление рисками, экономия кормовых затрат

---

## Резюме (Abstract)

**Контекст:** Традиционное линейное программирование (LP) для расчёта рационов использует фиксированные значения питательности и цен, игнорируя вариабельность и неопределённость. Это приводит к субоптимальным решениям в реальных условиях.

**Цель:** Разработать модель стохастического программирования (SP) для формулирования рационов, учитывающую вариабельность цен кормов и питательности, и сравнить её с традиционным LP.

**Методы:** Двухэтапная модель стохастического программирования с рекурсией. Сценарии вариабельности цен и питательности на основе исторических данных (2018-2022). Сравнение с LP по стоимости, риску и робастности.

**Результаты:** 
- Средняя экономия: 8-12% по сравнению с LP
- Снижение вариабельности стоимости на 35%
- Улучшение робастности рационов (меньше нарушений ограничений)
- ROI внедрения: 250-400%

**Выводы:** Стохастическое программирование превосходит традиционный LP по экономичности и надёжности. Внедрение оправдано для крупных хозяйств.

## 2. KEY CLAIMS (ключевые утверждения)

### Claim 1: LP даёт субоптимальные решения при вариабельности
- **Утверждение:** Традиционное линейное программирование, использующее средние значения, даёт рационы, которые в 35% случаев превышают бюджет или нарушают ограничения при реальной вариабельности
- **Evidence:** Симуляция 1000 сценариев с реальными данными
- **Confidence:** high
- **Цитата:** "Traditional LP formulations violate budget or nutrient constraints in 35% of scenarios when real-world variability is considered"

### Claim 2: SP снижает стоимость на 8-12%
- **Утверждение:** Стохастическое программирование даёт рационы, которые в среднем на 8-12% дешевле LP-рационов при сохранении всех ограничений
- **Evidence:** Сравнение 50 рационов, рассчитанных обоими методами
- **Confidence:** high
- **Цитата:** "Stochastic programming formulations reduced feed costs by 8 to 12% compared to deterministic LP while maintaining all nutritional constraints"

### Claim 3: SP улучшает робастность на 35%
- **Утверждение:** Рационы SP имеют на 35% меньше нарушений ограничений при вариации цен и питательности

- **Утверждение:** Двухэтапная модель SP с рекурсией обеспечивает оптимальное соотношение точности и вычислительной сложности для практического применения

## 3. Введение (Introduction)

### 3.1. Полный текст введения [перевод]

**Контекст:**

Формулирование рационов методом наименьшей стоимости (least-cost formulation) — стандартная практика в молочном скотоводстве. Традиционно используется линейное программирование (LP), минимизирующее стоимость при соблюдении ограничений по питательности.

**Проблема LP:**

1. **Детерминированность:** LP использует фиксированные значения цен и питательности
2. **Игнорирование вариабельности:** Цены кормов варьируют на 10-30%
3. **Питательность варьирует:** CV 5-15% для основных кормов
4. **Результат:** Расчётные рационы часто не работают в реальности

**Последствия:**
- Перерасход бюджета
- Нарушение ограничений
- Необходимость корректировок
- Субоптимальная экономика

**Решение:**

Стохастическое программирование (Stochastic Programming, SP) — метод оптимизации, явно учитывающий неопределённость через сценарии.

**Цель исследования:**

1. Разработать SP-модель для формулирования рационов
2. Сравнить с традиционным LP
3. Оценить экономические преимущества
4. Предложить практическую реализацию

**Ключевое сообщение:**

> "Неопределённость — не враг, если её правильно учитывать. Стохастическое программирование превращает вариабельность из проблемы в управляемый фактор."

### 3.2. Ключевые аргументы автора

**Тезис 1: Средние значения — ловушка**
- LP работает со средними
- Реальность — распределение
- Средний рацион ≠ рацион для средней ситуации

**Тезис 2: Сценарии лучше точечных оценок**
- SP использует множество сценариев
- Каждый сценарий — возможное будущее
- Оптимизация по ожиданию

**Тезис 3: Вычислительная сложность преодолима**
- SP сложнее LP, но решаемо
- Современные компьютеры справляются
- Облачные вычисления доступны

## 4. МАТЕРИАЛЫ И МЕТОДЫ (Materials and Methods)

### 4.1. Общее описание

**Тип:** Mathematical modeling + simulation  
**Данные:** Исторические цены кормов 2018-2022 (USDA, 5 лет)  
**Период:** Анализ вариабельности + моделирование  
**Инструменты:** Python (PuLP, Pyomo), Gurobi solver

### 4.2. Модель линейного программирования (LP)

**Целевая функция:**
```
Minimize: Σ(c_i × x_i)

Где:
c_i = цена корма i (фиксированная)
x_i = количество корма i в рационе
```

**Ограничения:**
- Энергия (NEL): минимум 1.75 Mcal/kg
- Белок (CP): 16-18%
- NDF: 28-35%
- Минералы: Ca, P, Mg в норме
- Inclusion rates: min/max для каждого корма

### 4.3. Модель стохастического программирования (SP)

**Двухэтапная модель с рекурсией:**

**Первый этап (здесь и сейчас):**
```
Minimize: Σ(c_i × x_i) + E[Q(x, ξ)]

Где:
c_i = ожидаемая цена
x_i = решение (рацион)
Q(x, ξ) = рекурсия (штраф за нарушения)
ξ = случайные параметры (цены, питательность)
```

**Второй этап (сценарии):**
```
Q(x, ξ) = Minimize: Σ(штраф за нарушения)

Для каждого сценария s:
- Реальная цена c_i^s
- Реальная питательность a_ji^s
- Проверка ограничений
- Расчёт штрафов
```

**Сценарии:**
- 50 сценариев цен (исторические данные)
- 30 сценариев питательности (NIRS вариабельность)
- Всего: 1500 комбинированных сценариев

## 10. FAQ

### Q1: Что такое стохастическое программирование простыми словами?
**A:** Это метод оптимизации, который учитывает неопределённость. Вместо одной "правильной" цены корма он использует множество сценариев (высокая цена, низкая цена, средняя) и находит решение, которое работает лучше всего в среднем по всем сценариям.

### Q2: Нужен ли математик для внедрения?
**A:** Для разработки модели — да. Для использования — нет. Современные программы имеют интерфейсы, где поставщик настраивает модель, а пользователь просто получает результаты.

### Q3: Сколько сценариев нужно?
**A:** Для практики достаточно 50-100 сценариев. Больше — лучше точность, но дольше расчёт. Исследование показывает, что 50 сценариев дают 95% эффекта от 1000.

### Q4: Работает ли при отсутствии исторических данных?
**A:** Можно использовать ожидаемую вариабельность из литературы (CV 15-25% для зерна). Это лучше, чем игнорировать вариабельность вообще, но хуже, чем собственные данные.

### Q5: Можно ли использовать для других животных?
**A:** Да, методология универсальна. Уже применяется для свиней, птицы, рыб. Требуется только адаптация ограничений и данных.
