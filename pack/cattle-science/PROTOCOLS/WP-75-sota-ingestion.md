# WP-75: SoTA Ingestion Protocol

> **Протокол:** WP-75  
> **Название:** SoTA Ingestion — погружение State of the Art  
> **Версия:** 1.3  
> **Базируется на:** CS.SOTA.001 (Lauber 2025)  
> **Автор:** PACK-cattle-science  
> **Дата:** 2026-03-14

---

## Назначение

Протокол WP-75 стандартизирует процесс создания SoTA (State of the Art) в PACK-cattle-science. Обеспечивает:
- Единое качество всех SoTA
- Эффективное использование времени (4ч/сессия)
- Совместимость с ArchGate и FPF
- Готовность к лекциям и практическому применению

---

## ⚠️ Параметры сессии: ТОЛЬКО ГЛУБОКО

| Параметр | Значение |
|----------|----------|
| **Длительность** | 4 часа (стандарт) или 8 часов (расширенная) |
| **Целевое количество** | **1 статья за 4 часа** |
| **Время на статью** | **~4 часа (ГЛУБОКО)** |
| **Формат выхода** | Полный SoTA v1.3 (ВСЕГДА ГЛУБОКО) |
| **Минимальный ArchGate** | **10/10** |
| **Режим** | **ТОЛЬКО ГЛУБОКО — без исключений** |

### 🔴 Критично: Обновление индекса ПОСЛЕ КАЖДОЙ СТАТЬИ

**Правило:** После завершения каждой SoTA (до перехода к следующей) — немедленно обновить `CS.MAP.001-sota-index.md`.

**Чек-лист индекс-чекпоинта:**
- [ ] Добавлена в "Полный список SoTA"
- [ ] Добавлена в "По структуре папок"
- [ ] Обновлены счётчики файлов
- [ ] Обновлена дата
- [ ] Проверка поиском (Ctrl+F находит по ID)

**Последствия пропуска:**
- SoTA становится "невидимой"
- Риск дублирования ID
- Некорректная оценка ArchGate
- Неполный статус WP-75

## Фазы протокола

### Фаза 0: Подготовка (до сессии)

**За 1 день:**
- [ ] Создать список 10-15 статей для обработки
- [ ] Проверить доступность (open access preferred)
- [ ] Скачать PDF всех статей
- [ ] Подготовить рабочее пространство

**Инструменты:**
- Zotero / Mendeley для управления ссылками
- Доступ к PubMed, Google Scholar
- Шаблон: `TEMPLATES/SOTA-TEMPLATE.md`

---

### Фаза 1: Открытие (15 мин)

**Цель:** Настройка окружения, проверка готовности

**Действия:**
1. Открыть индекс: `CS.MAP.001-sota-index.md`
2. Определить следующий свободный ID (CS.SOTA.XXX)
3. Создать папку: `06-sota/[area]/CS.SOTA.XXX-[краткое-название]/`
4. Скопировать шаблон: `cp TEMPLATES/SOTA-TEMPLATE.md CS.SOTA.XXX.md`
5. Заполнить YAML frontmatter (базовые поля)

**Выход:**
- Создана структура папок
- Готовый шаблон с метаданными

---

### Фаза 2: Работа (3 часа 20 мин)

#### Подфаза 2.1: Быстрое погружение (15 мин/статья × 10 = 2.5 часа)

**Для каждой статьи:**

1. **Чтение Abstract** (2 мин)
   - Контекст → Цель → Методы → Результаты → Выводы

2. **Сканирование структуры** (3 мин)
   - Introduction: проблема, гипотезы
   - M&M: тип исследования, ключевые параметры
   - Results: главные графики/таблицы
   - Conclusions: ключевые сообщения

3. **Заполнение SoTA** (10 мин)
   - Перевести Abstract
   - Перевести Introduction (кратко)
   - Описать 2-3 главных результата
   - Перевести Conclusions
   - Добавить 3-5 тегов
   - Найти 2-3 связанных SoTA

**Формат:** Упрощённый (без медиа-инвентаря, без FAQ, без лекций)

#### Подфаза 2.2: Глубокая проработка (50 мин для 2-3 ключевых статей)

**Выберите 2-3 самые важные статьи из 10:**

1. **Полный контент** (20 мин)
   - Полный перевод Introduction
   - Детальное описание M&M
   - Все результаты с цифрами

2. **Критический анализ** (15 мин)
   - Сильные/слабые стороны
   - Сравнение с другими исследованиями
   - Применимость к России

3. **Практика** (10 мин)
   - Алгоритм применения
   - 5-7 FAQ
   - Структура калькулятора

4. **Лекции** (5 мин)
   - Медиа-инвентарь для главных графиков
   - Ключевые комментарии

---

### Фаза 3: Закрытие (25 мин)

#### Подфаза 3.1: Оценка качества (10 мин)

**Для каждого SoTA проверить ArchGate:**

| Критерий | Балл | Минимум |
|----------|------|---------|
| Структура | 0-2 | ≥1 |
| Метаданные | 0-2 | ≥1 |
| Связи | 0-2 | ≥1 |
| Полнота | 0-2 | ≥1 |
| Источники | 0-2 | ≥2 |
| Актуальность | 0-2 | ≥2 |
| **Итого** | **0-20** | **≥8** |

**Если <8:** Добавить недостающие разделы или отметить для доработки

#### Подфаза 2.9: Индекс-чекпоинт (АВТОМАТИЗИРОВАН)

> **Автоматизация:** Индекс обновляется автоматически при коммите через git hooks.

**Настройка (один раз):**
```bash
git config core.hooksPath .githooks
```

**Процесс:**
1. Завершить SoTA (Phase 6)
2. `git add` и `git commit` как обычно
3. Hook автоматически:
   - Извлечёт метаданные из YAML
   - Добавит SoTA в индекс
   - Обновит счётчики и дату
   - Добавит индекс в коммит

**Ручное обновление (если нет hooks):**
```bash
bash scripts/add-sota-to-index.sh pack/cattle-science/06-sota/[area]/CS.SOTA.XXX-*.md
```

**Проверка:**
```bash
bash scripts/verify-sota-index.sh
```

---

#### Подфаза 3.2: Финальная проверка индекса (5 мин)

Проверить полноту `CS.MAP.001-sota-index.md`:
- Все SoTA сессии отображаются корректно
- Связи между источниками добавлены
- Счётчики файлов актуальны

#### Подфаза 3.3: Фиксация (10 мин)

- [ ] Заполнить журнал обработки в каждом SoTA
- [ ] git add .
- [ ] git commit -m "WP-75: Add X SoTA articles (YYYY-MM-DD)"
- [ ] (опционально) git push

## Критерии выбора "ключевых" статей

Статья считается ключевой, если:

1. **Высокий импакт-фактор** журнала (JDS, JDS Communications)
2. **Новизна** — первое исследование в области
3. **Практическая ценность** — можно сразу применить
4. **Связность** — связывает несколько тем
5. **Цитируемость** — часто упоминается в других работах

**Пример из WP-75 (2026-03-14):**
- Ключевая: CS.SOTA.001 (Lauber 2025) — первая экономическая оптимизация IEP
- Ключевая: CS.SOTA.015 (Fricke 2023) — High Fertility Cycle
