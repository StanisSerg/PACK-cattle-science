# CS.FM.001: Завышение 21-дневного индекса стельности

> Расчёт 21-d PR по одной точке данных или короткому периоду даёт завышенную оценку, что приводит к неверным управленческим решениям.

---

## Описание

### Что происходит

Менеджер фермы рассчитывает 21-дневный индекс стельности (21-d PR) по данным за 1-2 месяца или по небольшой выборке коров. Полученное значение (например, 32%) кажется хорошим, и принимается решение не менять репродуктивную программу. Однако при расчёте за 6-12 месяцев реальный 21-d PR оказывается значительно ниже (например, 24%).

**Типичный сценарий:**
1. Весной (март-апрель) фертильность выше
2. Расчёт 21-d PR за март-апрель показывает 30%
3. Решение: "Всё хорошо, не трогаем программу"
4. Летом (июль-август) тепловой стресс снижает фертильность
5. Годовой 21-d PR = 22%
6. Экономические потери от запоздалой коррекции

### Почему это проблема

**Последствия:**
- Ложное чувство безопасности
- Отсрочка необходимых улучшений
- Потери от неэффективной репродуктивной программы
- Упущенная экономия

**Экономический ущерб:**
- Разница 21-d PR 8% (30% → 22%) = ~$50-100/корова/год
- Для стада 500 коров = $25,000-50,000/год упущенной прибыли

## Признаки (Indicators)

| Признак | Как обнаружить | Порог тревоги |
|---------|----------------|---------------|
| Расчёт по <3 месяцам | Проверить период расчёта | <90 дней — критично |
| Выборка <100 коров | Проверить n в расчёте | <100 — ненадёжно |
| Сезонная вариация >5% | Сравнить месяцы | >5% — требует сглаживания |
| Расчёт "в уме" | Опросить, как считают | Без формулы — ошибка |

---

## Причины (Root Causes)

### 1. Сезонность фертильности
- **Механизм:** Тепловой стресс летом снижает PR; оптимальные условия весной повышают
- **Условия:** Регионы с выраженной сезонностью, отсутствие охлаждения

### 2. Небольшая выборка
- **Механизм:** При n<100 случайные колебания дают большую погрешность
- **Условия:** Малые стада, расчёт по подгруппам

### 3. Выбросы (outliers)
- **Механизм:** Один удачный месяц искажает картину
- **Условия:** Короткий период расчёта

### 4. Неправильная формула
- **Механизм:** Путаница между Service Rate, Conception Rate и 21-d PR
- **Условия:** Недостаточное обучение персонала

---

## Предотвращение (Prevention)

### Процедурные меры
- [ ] Стандарт: расчёт 21-d PR только по скользящему среднему за 3-6 месяцев
- [ ] Минимальная выборка: n ≥ 100 коров
- [ ] Ежемесячный расчёт с отслеживанием тренда
- [ ] Сезонная коррекция: сравнение с аналогичным периодом прошлого года

### Технические меры
- [ ] Автоматический расчёт в DC305/DairyComp с настройкой "rolling 90-day"
- [ ] Дашборд с графиком тренда 21-d PR
- [ ] Алерт при изменении >5% за месяц

---

## Обнаружение (Detection)

### Ранние признаки
- Разница между месячным и квартальным 21-d PR > 5%
- Необъяснимый скачок 21-d PR в одном месяце

### Методы диагностики
| Метод | Частота | Ответственный |
|-------|---------|---------------|
| Проверка периода расчёта | Ежемесячно | Ветеринар/консультант |
| Сравнение 1/3/6/12-месячных PR | Ежеквартально | Менеджер |
| Аудит методологии расчёта | Ежегодно | Внешний консультант |

---

## Реагирование (Response)

### Немедленные действия
1. Пересчитать 21-d PR за 3, 6 и 12 месяцев
2. Построить график тренда
3. Определить сезонный паттерн

### Корректирующие действия
1. Установить стандарт: только rolling 90-day average
2. Обучить персонал правильной методологии
3. Настроить ПО на автоматический расчёт
4. Включить сезонную коррекцию в планирование

## Примеры из практики

### Пример 1: Весенний оптимизм
**Ситуация:** Ферма в Висконсине рассчитала 21-d PR = 31% по апрелю. Решили не менять программу.
**Последствия:** Годовой 21-d PR оказался 23%. Calving interval увеличился с 13.2 до 14.1 месяцев.
**Урок:** Всегда использовать минимум 3-месячное скользящее среднее.

### Пример 2: Малое стадо
**Ситуация:** Ферма на 120 коров рассчитывала 21-d PR ежемесячно. Значения колебались от 15% до 45%.
**Последствия:** Принимались контрпродуктивные решения (меняли протоколы каждый месяц).
**Урок:** Для малых стад использовать 6-месячное среднее или объединять данные с похожими фермами.
