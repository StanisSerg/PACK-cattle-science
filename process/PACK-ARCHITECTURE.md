## Философия архитектуры

PACK-cattle-science построен по многоуровневой архитектуре, отделяющей:
- **Фундаментальные знания** (Core) от **практического применения** (Applications)
- **Теорию** (SoTA) от **методов** (Methods)
- **Общие принципы** от **конкретных ситуаций**

Это позволяет:
- Быстро находить нужную информацию по ролям
- Стандартизировать практику через Methods
- Адаптировать знания под конкретные задачи через Applications
- Поддерживать актуальность без полной переработки

## Типы сущностей

### 1. Core (`type: core`)

**Назначение:** Фундаментальные концепции, на которые ссылаются все остальные уровни

**Характеристики:**
- ID: `CS.CORE.XXX`
- Папка: `core/`
- Стабильность: Высокая (изменяются редко)
- Связи: Используются всеми другими уровнями

**Примеры:** Cow, Milk, Rumen, Metabolism, Reproductive System, Immunity

### 2. SoTA (`type: sota`)

**Назначение:** Научные знания, исследования, рекомендации

**Характеристики:**
- ID: `CS.SOTA.XXX`
- Папки: `feeding/`, `reproduction/`, `health/`
- Стабильность: Средняя (обновляются с новыми исследованиями)
- Связи: Зависят от Core, используются Methods

**Подтипы:**
- `subtype: book-chapter` — главы книг (NASEM)
- `subtype: research-paper` — исследовательские статьи
- `subtype: synthesis` — обобщения

### 3. Methods (`type: method`)

**Назначение:** Практические инструкции, SOP, расчётные алгоритмы

**Характеристики:**
- ID: `CS.METHOD.XXX`
- Папка: `methods/`
- Стабильность: Высокая (меняются только при улучшении процесса)
- Связи: Реализуют SoTA, используются Applications

**Обязательные секции:**
- Цель
- Необходимое оборудование
- Входные данные
- Пошаговая инструкция
- Чек-лист качества
- Типичные ошибки

### 4. Applications (`type: application`)

**Назначение:** Готовые протоколы для конкретных ситуаций

**Характеристики:**
- ID: `CS.APP.XXX`
- Папка: `applications/`
- Стабильность: Средняя (адаптируются под условия)
- Связи: Используют Methods и SoTA

**Обязательные секции:**
- Описание ситуации
- Цели
- Входные данные
- Пошаговый протокол
- Мониторинг
- Чек-листы

### 5. Map (`type: map`)

**Назначение:** Навигация и индексация

**Характеристики:**
- ID: `CS.MAP.XXX`
- Папка: корень `06-sota/`
- Стабильность: Обновляется при изменении структуры

## Связи между уровнями

### Принцип зависимостей

```
Applications → Methods → SoTA → Core
     ↑            ↑         ↑
     └────────────┴─────────┘
           Зависят от
```

**Верхние уровни зависят от нижних, но не наоборот:**
- Core не знает о существовании Methods, SoTA, Applications
- SoTA ссылается на Core
- Methods реализуют SoTA
- Applications используют Methods

### Типы связей

| Тип | Описание | Пример |
|-----|----------|--------|
| `depends_on` | Зависит от знаний | Methods depends_on SoTA |
| `implements` | Реализует в практику | Methods implements SoTA |
| `uses` | Использует | Applications uses Methods |
| `extends` | Расширяет | SoTA extends Core |

## Поиск и навигация

### По ролям

| Роль | Начать с | Использовать |
|------|----------|--------------|
| **Технолог** | Applications | Methods для расчётов |
| **Ветеринар** | Health SoTA | Methods диагностики |
| **Репродуктолог** | Reproduction SoTA | Methods синхронизации |
| **Студент** | Core | SoTA для углубления |
| **Менеджер** | Central Hub | Applications для KPI |

### По задаче

| Задача | Путь |
|--------|------|
| Составить рацион | Applications → Methods |
| Обучить персонал | Methods (SOP) |
| Проверить нормы | SoTA (NASEM) |
| Понять физиологию | Core |

## Процесс добавления новых сущностей

### Алгоритм принятия решения

```
Новый контент
    ↓
Это фундаментальная сущность?
    ├── ДА → core/ (CS.CORE.XXX)
    └── НЕТ
        ↓
    Это практический протокол?
        ├── ДА → applications/ (CS.APP.XXX)
        └── НЕТ
            ↓
        Это метод/алгоритм?
            ├── ДА → methods/ (CS.METHOD.XXX)
            └── НЕТ
                ↓
            Это научное знание?
                ├── ДА → feeding/|reproduction/|health/ (CS.SOTA.XXX)
                └── НЕТ → Определить вручную
```

### Проверка перед созданием

- [ ] Правильный `type` в YAML
- [ ] Правильный ID по диапазону
- [ ] Правильная папка
- [ ] Ссылки на зависимости (target_sota, target_methods)
- [ ] Обновление Central Hub
