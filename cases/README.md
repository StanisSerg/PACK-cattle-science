# Cases / Кейсы

> Сырые факты с фермы.  
> Формат: вход → действие → результат

## Структура кейса

Каждый кейс — это **одно наблюдение** или **одна ситуация**:
- Проблема с конкретным животным
- Групповое событие
- Эксперимент/вмешательство

## Конвейер

```
CASE → DL → RULE
  ↓      ↓      ↓
Факты  Решение  Знание
```

**CASE сам по себе — ещё не система.**  
Это сырой материал для извлечения решений.

## Шаблон

Использовать: [TEMPLATE-CASE.md](TEMPLATE-CASE.md)

## Именование файлов

```
CASE-NNN-{краткое-описание}.md

Примеры:
- CASE-001-bhb-threshold.md
- CASE-002-mastitis-early-detection.md
- CASE-003-ration-transition-failure.md
```

## Связь с другими слоями

```
cases/CASE-001.md
    ↓ (порождает)
decisions/DL-001.md
    ↓ (формализуется в)
pack/rules/RULE-001.md
    ↓ (применяется к следующему)
cases/CASE-NNN.md
```

**Замкнутый цикл обучения.**
