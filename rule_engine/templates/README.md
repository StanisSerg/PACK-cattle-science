# Шаблоны CSV для загрузки данных фермы

> **Для:** WP-93 Decision Layer / PACK-cattle-science Rule Engine  
> **Формат:** CSV с разделителем `,` и десятичной точкой `.`

---

## Три уровня шаблонов

| Уровень | Файл | Что работает | Кол-во полей |
|---------|------|--------------|--------------|
| **L0** | `herd-data-template-L0.csv` | Продуктивность + репродукция + базовый скрининг метаболизма | 10 |
| **L1** | `herd-data-template-L1.csv` | + Диагностика метаболизма (кетоз, гипокальceмия, метритис) | 16 |
| **L2** | `herd-data-template-L2.csv` | + Мастит, хромота, полный отсев (все 12 правил) | 23 |

---

## L0 — Минимальный шаблон

### Поля

| Поле | Тип | Описание | Пример |
|------|-----|----------|--------|
| `cow_id` | string | Уникальный номер коровы | `COW-001` |
| `pen` | string | Группа / дойный зал | `1` |
| `dim` | int | День лактации | `45` |
| `parity` | int | Парность (1 = первотёлка) | `2` |
| `milk_yield` | float | Текущий дневной удой, кг | `24.0` |
| `pik_milk` | float | Пиковый удой (текущей/предыдущей лактации), кг | `38.0` |
| `pik_day` | int | День достижения пика (обычно 35–50) | `40` |
| `m305` | float | Удой за 305 дней (прогноз/факт) | `8200` |
| `insemination_count` | int | Количество осеменений в текущую лактацию | `1` |
| `reproduction_status` | string | `open` / `bred` / `pregnant` / `dry` / `cull` | `bred` |

### Активные правила

- **RULE-012** — Отклонения удоя (критические / умеренные / гиперпродуктивность / персистентность)
- **RULE-010** — Отсев (частично, через продуктивность + осеменения)
- **CS.WP.002** — Отчёт по продуктивности
- **CS.WP.003** — Скрининговый отчёт по метаболизму (прокси через DIM + отклонение удоя)

### Важно

- `m305` — **ключевое поле**. Оно масштабирует herd-level MilkBot-кривую до индивидуальной нормы коровы. Без него ожидаемый удой считается грубо.
- `pik_milk` + `pik_day` — усиливают точность расчёта персистентности. Если неизвестны, можно поставить `pik_day = 45`, а `pik_milk ≈ m305 / 305 * 1.8`.

## L1 — Расширенный шаблон (метаболизм)

### Дополнительные поля

| Поле | Тип | Описание | Пример | Правило |
|------|-----|----------|--------|---------|
| `bhb` | float | BHB в крови/сыворотке, ммоль/л | `1.2` | RULE-002 (SCK) |
| `temperature` | float | Ректальная температура, °C | `39.2` | RULE-006 (метритис) |
| `retained_placenta_hours` | int | Задержка последа, часы (0 = нет) | `24` | RULE-006 |
| `uterine_discharge_foul` | bool | Маточные выделения с неприятным запахом | `true` | RULE-006 |
| `clinical_milk_fever` | bool | Клиническая молочная лихорадка | `false` | RULE-005 |
| `calcium_mmol_l` | float | Кальций крови, ммоль/л | `2.0` | RULE-005 |

### Активные правила

Все из L0, плюс:
- **RULE-002** — Субклинический кетоз (BHB ≥ 1.2, DIM 3–14)
- **RULE-005** — Гипокальceмия / молочная лихорадка
- **RULE-006** — Метритис / задержка последа
- **CS.WP.003** — Переходит из «скринингового» в «диагностический"

### Пороговые значения (для справки)

| Показатель | Норма | Тревога | Критично |
|------------|-------|---------|----------|
| BHB | < 1.0 | 1.0–1.2 | ≥ 1.2 |
| Температура | 38.0–39.0 | 39.0–39.5 | ≥ 39.5 |
| Ca | ≥ 2.0 | 1.8–2.0 | < 1.8 |
| RP | < 12 ч | 12–24 ч | > 24 ч |

## L2 — Полный шаблон (все 12 правил)

### Дополнительные поля

| Поле | Тип | Описание | Пример | Правило |
|------|-----|----------|--------|---------|
| `scc` | int | Соматические клетки, тыс/мл | `150000` | RULE-011 |
| `cmt_score` | int | CMT-тест (1–3) | `2` | RULE-011 |
| `mastitis_cases_90d` | int | Случаев мастита за 90 дней | `1` | RULE-010, RULE-011 |
| `locomotion_score` | int | Шкала хромоты (1–3) | `2` | RULE-009, RULE-010 |
| `treatment_cost_90d` | int | Затраты на лечение за 90 дн, ₽ | `5000` | RULE-010 |
| `dmi_actual` | float | Фактический съем корма, кг | `22.0` | Усиливает все метаболические |
| `bcs` | float | Условная оценка упитанности (1–5) | `3.0` | RULE-001, RULE-002 |

### Активные правила

Все из L1, плюс:
- **RULE-009** — Хромота (Locomotion Score ≥ 2)
- **RULE-011** — Мастит (SCC > 400k, CMT ≥ 3, клинические признаки)
- **RULE-010** — Полный отсев (метаболизм + мастит + хромота + затраты)

## Соответствие полей и Data.csv предприятия

Если у вас уже есть выгрузка вида `Data.csv` (ОАО МТК):

| Ваше поле (Data.csv) | Наше поле | Примечание |
|----------------------|-----------|------------|
| `№` | `cow_id` | |
| `Pen` | `pen` | |
| `Stat` | `reproduction_status` | `ТЕЧКА`→`open`, `ОСЕМ`→`bred`, `СТЕЛ`→`pregnant`, `БРАК`→`cull`, `ЯЛОВ`→`open` |
| `DIM` | `dim` | |
| `Lactno` | `parity` | |
| `Milk` | `milk_yield` | |
| `M305` | `m305` | |

### Чего не хватает в стандартной выгрузке

- Коровы с `DIM < 35` (критично для RULE-002/005/006)
- `insemination_count` (усиливает RULE-010)
- Лабораторные данные: BHB, Ca, температура
